# Greedy Pattern – Variation 4: Sorting & Ordering Greedy

## Core Idea

Sort the data first, then make the greedy choice.

Many greedy problems become easy only after sorting.

> **Sort → Greedy Choice → Optimal Answer**

---

## Recognition Signals

- Match two arrays
- Rearrange elements
- Maximize wins
- Assign resources
- Ordering matters
- Queue reconstruction
- Largest/Smallest arrangement
- Custom comparator

```text
Unsorted Input
      │
```

# Problem 1 — LC455 Assign Cookies ⭐⭐⭐

### Recognition

- Match two arrays
- Maximum satisfied children

### Intuition

Sort both arrays.

Give the smallest cookie that satisfies the least greedy child.

Never waste a larger cookie.

### Code

```java
class Solution {

    public int findContentChildren(int[] g, int[] s) {

        Arrays.sort(g);
        Arrays.sort(s);

        int child = 0;
        int cookie = 0;

        while(child < g.length && cookie < s.length){

            if(s[cookie] >= g[child])
                child++;

            cookie++;
        }

        return child;
    }
}
```

**Time:** O(N log N)

**Space:** O(1)

# Problem 2 — LC826 Most Profit Assigning Work ⭐⭐⭐⭐

### Recognition

- Jobs
- Workers
- Assignment
- Maximum profit

### Intuition

Sort jobs by difficulty.

Sort workers.

Maintain the maximum profit seen so far.

### Code

```java
class Solution {

    public int maxProfitAssignment(int[] difficulty,
                                   int[] profit,
                                   int[] worker) {

        int n = difficulty.length;

        int[][] jobs = new int[n][2];

        for(int i=0;i<n;i++){

            jobs[i][0]=difficulty[i];
            jobs[i][1]=profit[i];
        }

        Arrays.sort(jobs,
                (a,b)->a[0]-b[0]);

        Arrays.sort(worker);

        int answer=0;

        int bestProfit=0;

        int job=0;

        for(int ability:worker){

            while(job<n &&
                    jobs[job][0]<=ability){

                bestProfit=
                        Math.max(bestProfit,
                                jobs[job][1]);

                job++;
            }

            answer+=bestProfit;
        }

        return answer;
    }
}
```

**Time:** O(N log N)

**Space:** O(N)

# Problem 3 — LC870 Advantage Shuffle ⭐⭐⭐⭐

### Recognition

- Rearrange array
- Maximize wins

### Intuition

Sort nums1.

Sort indices of nums2.

If largest number can beat the opponent,
use it.

Otherwise sacrifice the smallest number.

### Code

```java
class Solution {

    public int[] advantageCount(int[] nums1,
                                int[] nums2) {

        Arrays.sort(nums1);

        int n=nums1.length;

        Integer[] idx=new Integer[n];

        for(int i=0;i<n;i++)
            idx[i]=i;

        Arrays.sort(idx,
                (a,b)->Integer.compare(
                        nums2[a],
                        nums2[b]));

        int left=0;
        int right=n-1;

        int[] ans=new int[n];

        for(int i=n-1;i>=0;i--){

            int index=idx[i];

            if(nums1[right]>nums2[index]){

                ans[index]=nums1[right--];
            }

            else{

                ans[index]=nums1[left++];
            }
        }

        return ans;
    }
}
```

**Time:** O(N log N)

**Space:** O(N)

# Problem 4 — LC406 Queue Reconstruction by Height ⭐⭐⭐⭐⭐

### Recognition

- Queue
- Position constraints
- Ordering

### Intuition

Sort by

- Height descending
- k ascending

Insert each person at index k.

### Code

```java
class Solution {

    public int[][] reconstructQueue(int[][] people) {

        Arrays.sort(people,(a,b)->{

            if(a[0]==b[0])
                return a[1]-b[1];

            return b[0]-a[0];
        });

        List<int[]> list=new ArrayList<>();

        for(int[] person:people){

            list.add(person[1],person);
        }

        return list.toArray(new int[list.size()][]);
    }
}
```

**Time:** O(N²)

**Space:** O(N)

# Problem 5 — LC179 Largest Number ⭐⭐⭐⭐⭐

### Recognition

- Rearrange numbers
- Largest number
- Custom comparator

### Intuition

Normal sorting fails.

Compare

```text
ab

vs

ba
```

If

```text
ab > ba
```

then

```text
a comes before b
```

### Code

```java
class Solution {

    public String largestNumber(int[] nums) {

        String[] arr=new String[nums.length];

        for(int i=0;i<nums.length;i++)
            arr[i]=String.valueOf(nums[i]);

        Arrays.sort(arr,
                (a,b)->(b+a).compareTo(a+b));

        if(arr[0].equals("0"))
            return "0";

        StringBuilder sb=new StringBuilder();

        for(String s:arr)
            sb.append(s);

        return sb.toString();
    }
}
```

**Time:** O(N log N)

**Space:** O(N)
