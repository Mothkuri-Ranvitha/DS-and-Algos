#  Rotated Sorted Array Problems

## Variation Definition

In Classical Binary Search:

```text
Array is completely sorted.
```

Example:

```java
[1,2,3,4,5,6,7]
```

In Rotated Array Problems:

```text
Array was originally sorted
but rotated at some pivot.
```

Example:

```java
[4,5,6,7,0,1,2]
```

The array is no longer globally sorted.

But:

```text
At least one half
is always sorted.
```

This observation is the entire pattern.

---

# Why Does This Variation Exist?

Original:

```java
[0,1,2,4,5,6,7]
```

Rotate at index 4:

```java
[4,5,6,7,0,1,2]
```

Question:

```text
Can we still use Binary Search?
```

Yes.

Because:

```text
One side remains sorted.
```

---

# Core Intuition

For every mid:

```java
mid
```

One of these must be sorted:

```text
Left Half

OR

Right Half
```

Identify the sorted half.

Check whether target belongs there.

Discard the other half.

# Problem 1

## LC33 - Search in Rotated Sorted Array

### Problem

Given:

```java
nums = [4,5,6,7,0,1,2]

target = 0
```

Return:

```java
4
```

---

# Intuition

At every step:

Determine:

```text
Which half is sorted?
```

---

Example:

```java
[4,5,6,7,0,1,2]
```

mid:

```java
7
```

Left Half:

```java
[4,5,6,7]
```

is sorted.

Now ask:

```text
Does target belong
inside this sorted range?
```

If yes:

```text
Search Left
```

Else:

```text
Search Right
```

---

# Dry Run

Input:

```java
[4,5,6,7,0,1,2]

target = 0
```

---

## Iteration 1

```java
left = 0
right = 6

mid = 3
```

```java
nums[mid] = 7
```

Left side sorted:

```java
4 5 6 7
```

Check:

```java
4 <= 0 <= 7
```

False.

Search right.

```java
left = 4
```

---

## Iteration 2

```java
left = 4
right = 6

mid = 5
```

```java
nums[mid] = 1
```

Right side sorted:

```java
1 2
```

Target not there.

Search left.

```java
right = 4
```

---

## Iteration 3

```java
mid = 4
```

```java
nums[mid] = 0
```

Found.

Return:

```java
4
```

---

# Code

```java
class Solution {

    public int search(
            int[] nums,
            int target) {

        int left = 0;
        int right = nums.length - 1;

        while(left <= right){

            int mid =
                left + (right-left)/2;

            if(nums[mid] == target)
                return mid;

            // Left sorted
            if(nums[left] <= nums[mid]){

                if(nums[left] <= target
                        &&
                        target < nums[mid]){

                    right = mid - 1;
                }
                else{

                    left = mid + 1;
                }
            }

            // Right sorted
            else{

                if(nums[mid] < target
                        &&
                        target <= nums[right]){

                    left = mid + 1;
                }
                else{

                    right = mid - 1;
                }
            }
        }

        return -1;
    }
}
```

---

# Complexity

```text
Time  : O(log N)

Space : O(1)
```

# Problem 2

## LC153 - Find Minimum in Rotated Sorted Array

### Problem

Given:

```java
[4,5,6,7,0,1,2]
```

Return:

```java
0
```

---

# Key Observation

Minimum element is the:

```text
Pivot Point
```

Example:

```java
4 5 6 7 0 1 2
        ^
```

---

# Intuition

Compare:

```java
nums[mid]
```

with:

```java
nums[right]
```

---

Case 1

```java
nums[mid] > nums[right]
```

Minimum lies right.

Move:

```java
left = mid + 1
```

---

Case 2

```java
nums[mid] <= nums[right]
```

Minimum lies left including mid.

Move:

```java
right = mid
```

---

# Dry Run

```java
[4,5,6,7,0,1,2]
```

---

## Iteration 1

```java
mid = 3

nums[mid] = 7

nums[right] = 2
```

```java
7 > 2
```

Minimum right.

```java
left = 4
```

---

## Iteration 2

```java
left = 4
right = 6

mid = 5
```

```java
nums[mid] = 1

nums[right] = 2
```

```java
1 <= 2
```

Minimum left.

```java
right = 5
```

---

## Iteration 3

```java
left = 4
right = 5

mid = 4
```

```java
nums[mid] = 0
```

Minimum found.

Loop ends.

---

# Code

```java
class Solution {

    public int findMin(int[] nums) {

        int left = 0;
        int right = nums.length - 1;

        while(left < right){

            int mid =
                left + (right-left)/2;

            if(nums[mid] > nums[right]){

                left = mid + 1;
            }
            else{

                right = mid;
            }
        }

        return nums[left];
    }
}
```

---

# Complexity

```text
Time  : O(log N)

Space : O(1)
```

# Problem 3

## LC81 - Search in Rotated Sorted Array II

### Difference From LC33

LC33:

```text
No duplicates.
```

LC81:

```text
Duplicates allowed.
```

Example:

```java
[1,0,1,1,1]
```

Now:

```java
nums[left]
==
nums[mid]
==
nums[right]
```

can happen.

We cannot identify the sorted side.

---

# Trick

When:

```java
nums[left]
==
nums[mid]
==
nums[right]
```

Shrink both sides.

```java
left++;
right--;
```

Then continue normal LC33 logic.

---

# Code

```java
class Solution {

    public boolean search(
            int[] nums,
            int target) {

        int left = 0;
        int right = nums.length - 1;

        while(left <= right){

            int mid =
                left + (right-left)/2;

            if(nums[mid] == target)
                return true;

            if(nums[left] == nums[mid]
                    &&
                    nums[mid] == nums[right]){

                left++;
                right--;
            }

            else if(nums[left] <= nums[mid]){

                if(nums[left] <= target
                        &&
                        target < nums[mid]){

                    right = mid - 1;
                }
                else{

                    left = mid + 1;
                }
            }

            else{

                if(nums[mid] < target
                        &&
                        target <= nums[right]){

                    left = mid + 1;
                }
                else{

                    right = mid - 1;
                }
            }
        }

        return false;
    }
}
```
