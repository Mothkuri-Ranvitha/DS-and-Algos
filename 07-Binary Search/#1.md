# 01 - Classical Binary Search

## Variation Definition

This is the purest form of Binary Search.

Goal:

```text
Find the target element in a sorted array.
```

We repeatedly divide the search space into two halves and eliminate one half.

---

# Recognition Signals

Think Classical Binary Search when you see:

* Sorted Array
* Search Target
* Find Element
* Return Index

Keywords:

```text
Search

Target

Sorted Array

Index
```

---

# Core Intuition

Suppose:

```text
[1,3,5,7,9,11]
```

Target:

```text
9
```

Instead of checking:

```text
1 → 3 → 5 → 7 → 9
```

Check middle:

```text
7
```

If:

```text
9 > 7
```

discard left half.

Search only:

```text
[9,11]
```

Every comparison removes half the array.

---

# Generic Template

```java
int left = 0;
int right = nums.length - 1;

while(left <= right){

    int mid =
        left + (right-left)/2;

    if(nums[mid] == target)
        return mid;

    else if(nums[mid] < target)
        left = mid + 1;

    else
        right = mid - 1;
}

return -1;
```

# Problem 1

## LC704 - Binary Search

### Problem

Given:

```java
int[] nums
```

sorted in ascending order.

Return index of target.

If not found:

```text
return -1
```

---

### Example

Input:

```java
nums = [-1,0,3,5,9,12]

target = 9
```

Output:

```java
4
```

---

# Intuition

Because array is sorted:

```text
Left Side < Mid < Right Side
```

After checking mid:

```text
Target smaller → move left

Target larger → move right
```

Half of the array becomes useless.

---

# Dry Run

Input:

```java
[-1,0,3,5,9,12]

target = 9
```

---

## Iteration 1

```java
left = 0
right = 5

mid = 2
```

```java
nums[mid] = 3
```

Since:

```java
9 > 3
```

Move right.

```java
left = 3
```

---

## Iteration 2

```java
left = 3
right = 5

mid = 4
```

```java
nums[mid] = 9
```

Found.

Return:

```java
4
```

---

# Code

```java
class Solution {

    public int search(int[] nums, int target) {

        int left = 0;
        int right = nums.length - 1;

        while(left <= right){

            int mid =
                left + (right-left)/2;

            if(nums[mid] == target)
                return mid;

            else if(nums[mid] < target)
                left = mid + 1;

            else
                right = mid - 1;
        }

        return -1;
    }
}
```

---

# Complexity

```text
Time  : O(log N)

Space : O(1)
```

# Problem 2

## LC35 - Search Insert Position

### Problem

Given:

```java
nums = [1,3,5,6]
target = 2
```

Return:

```text
Index if found

Otherwise insertion position
```

Output:

```java
1
```

because:

```text
2 should be inserted before 3
```

---

# Intuition

Normal Binary Search only returns:

```text
Found / Not Found
```

Here we need:

```text
Where should target be inserted?
```

Observation:

When Binary Search ends:

```java
left
```

always points to the correct insertion position.

---

# Why?

Example:

```java
[1,3,5,6]

target = 2
```

Binary Search ends with:

```java
left = 1
right = 0
```

Position:

```java
1
```

which is exactly where 2 belongs.

---

# Dry Run

Input:

```java
[1,3,5,6]

target = 2
```

---

## Iteration 1

```java
left = 0
right = 3

mid = 1
```

```java
nums[mid] = 3
```

```java
2 < 3
```

Move left.

```java
right = 0
```

---

## Iteration 2

```java
left = 0
right = 0

mid = 0
```

```java
nums[mid] = 1
```

```java
2 > 1
```

Move right.

```java
left = 1
```

Loop ends.

Return:

```java
left
```

Answer:

```java
1
```

---

# Code

```java
class Solution {

    public int searchInsert(
            int[] nums,
            int target) {

        int left = 0;
        int right = nums.length - 1;

        while(left <= right){

            int mid =
                left + (right-left)/2;

            if(nums[mid] == target)
                return mid;

            else if(nums[mid] < target)
                left = mid + 1;

            else
                right = mid - 1;
        }

        return left;
    }
}
```
