# 06 - Peak / Mountain Array Problems

## Variation Definition

In previous Binary Search problems:

```text
Target exists somewhere.
```

We searched for:

```text
Target Value
Boundary
Answer
```

In Peak Problems:

```text
We search for a local maximum.
```

We don't care about sorted order.

We use the shape of the array.

---

# What is a Peak?

An element is a Peak if:

```java
nums[i] > nums[i-1]
AND
nums[i] > nums[i+1]
```

Example:

```java
[1,2,3,1]
```

Peak:

```java
3
```

---

# Why Binary Search Works?

Observation:

Suppose:

```java
nums[mid] < nums[mid+1]
```

Example:

```java
1 2 3 4 5
      ^
```

We are climbing uphill.

Peak must exist on the right.

Move:

```java
left = mid + 1
```

---

Suppose:

```java
nums[mid] > nums[mid+1]
```

Example:

```java
5 4 3 2 1
  ^
```

We are going downhill.

Peak must exist on the left side including mid.

Move:

```java
right = mid
```

---

# Core Intuition

Instead of checking:

```text
Target vs Mid
```

We check:

```text
Mid vs Mid+1
```

This tells us:

```text
Going Up

or

Going Down
```

Then eliminate half.

# Problem 1

## LC162 - Find Peak Element

### Problem

Given:

```java
nums = [1,2,3,1]
```

Return:

```java
2
```

because:

```java
nums[2] = 3
```

is a peak.

---

# Intuition

Compare:

```java
nums[mid]
```

and

```java
nums[mid+1]
```

---

Case 1

```java
nums[mid] < nums[mid+1]
```

Peak lies right.

Move:

```java
left = mid + 1
```

---

Case 2

```java
nums[mid] > nums[mid+1]
```

Peak lies left including mid.

Move:

```java
right = mid
```

---

# Dry Run

Input:

```java
[1,2,3,1]
```

---

Iteration 1

```java
left = 0
right = 3

mid = 1
```

```java
nums[mid] = 2
nums[mid+1] = 3
```

```java
2 < 3
```

Move right.

```java
left = 2
```

---

Iteration 2

```java
left = 2
right = 3

mid = 2
```

```java
nums[mid] = 3
nums[mid+1] = 1
```

```java
3 > 1
```

Move left.

```java
right = 2
```

Stop.

Answer:

```java
2
```

---

# Code

```java
class Solution {

    public int findPeakElement(int[] nums) {

        int left = 0;
        int right = nums.length - 1;

        while(left < right){

            int mid =
                left + (right-left)/2;

            if(nums[mid] < nums[mid+1]){

                left = mid + 1;
            }
            else{

                right = mid;
            }
        }

        return left;
    }
}
```

---

# Complexity

```text
Time  : O(log N)

Space : O(1)
```

# Problem 2

## LC852 - Peak Index in a Mountain Array

### Problem

A mountain array:

```java
[0,2,4,6,5,3,1]
```

Return:

```java
3
```

because:

```java
nums[3] = 6
```

is the peak.

---

# Observation

Mountain array has:

```text
Increasing Part

Peak

Decreasing Part
```

Example:

```java
0 2 4 6 5 3 1
      ^
```

---

# Intuition

Exactly same as LC162.

Compare:

```java
mid
```

and

```java
mid+1
```

---

If climbing:

```java
nums[mid] < nums[mid+1]
```

move right.

---

If descending:

```java
nums[mid] > nums[mid+1]
```

move left.

---

# Code

```java
class Solution {

    public int peakIndexInMountainArray(
            int[] arr) {

        int left = 0;
        int right = arr.length - 1;

        while(left < right){

            int mid =
                left + (right-left)/2;

            if(arr[mid] < arr[mid+1]){

                left = mid + 1;
            }
            else{

                right = mid;
            }
        }

        return left;
    }
}
```

---

# Complexity

```text
Time  : O(log N)

Space : O(1)
```

# Problem 3

## LC1095 - Find in Mountain Array

### Problem

Mountain Array:

```java
1 2 3 4 5 3 1
```

Target:

```java
3
```

Find index.

---

# Intuition

Step 1

Find Peak.

Use LC852.

```java
peak = 4
```

---

Step 2

Binary Search Left Half

```java
1 2 3 4 5
```

Ascending.

Normal Binary Search.

---

Step 3

If not found:

Binary Search Right Half

```java
5 3 1
```

Descending.

Reverse Binary Search.

---

# Algorithm

```text
Find Peak

Search Left Side

Search Right Side
```

---

# Code

```java
class Solution {

    public int findInMountainArray(
            int target,
            MountainArray mountainArr) {

        int n = mountainArr.length();

        int peak =
            findPeak(mountainArr);

        int left =
            binarySearchAsc(
                    mountainArr,
                    0,
                    peak,
                    target);

        if(left != -1)
            return left;

        return binarySearchDesc(
                mountainArr,
                peak + 1,
                n - 1,
                target);
    }

    private int findPeak(
            MountainArray arr){

        int left = 0;
        int right = arr.length()-1;

        while(left < right){

            int mid =
                left + (right-left)/2;

            if(arr.get(mid)
                    <
                    arr.get(mid+1))
                left = mid + 1;
            else
                right = mid;
        }

        return left;
    }

    private int binarySearchAsc(
            MountainArray arr,
            int left,
            int right,
            int target){

        while(left <= right){

            int mid =
                left + (right-left)/2;

            int value =
                arr.get(mid);

            if(value == target)
                return mid;

            if(value < target)
                left = mid + 1;
            else
                right = mid - 1;
        }

        return -1;
    }

    private int binarySearchDesc(
            MountainArray arr,
            int left,
            int right,
            int target){

        while(left <= right){

            int mid =
                left + (right-left)/2;

            int value =
                arr.get(mid);

            if(value == target)
                return mid;

            if(value > target)
                left = mid + 1;
            else
                right = mid - 1;
        }

        return -1;
    }
}
```


```text

```

```text
nums[mid] vs nums[mid+1]
```

will tell you which half contains the peak.
