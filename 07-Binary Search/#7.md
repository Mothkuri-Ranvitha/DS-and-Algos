# 08 - Missing / Duplicate / Unique Elements

## Variation Definition

This Binary Search variation is based on finding the position where an expected pattern breaks.

Unlike normal Binary Search:

```text
Find Target
```

we search for:

```text
Missing Number

Duplicate Number

Unique Element

Broken Pair Pattern
```

---

# Core Intuition

Most problems in this category have:

```text
Expected Pattern

↓

Broken Pattern

↓

Expected Pattern
```

or

```text
Correct Region

↓

Boundary

↓

Incorrect Region
```

Binary Search is used to find the boundary.

# Problem 1

# LC268 - Missing Number

## Problem Description

Given:

```java
nums = [0,1,3]
```

Numbers should be:

```java
0 1 2 3
```

Missing number:

```java
2
```

Return:

```java
2
```

---

## Intuition

Observe:

```java
index : 0 1 2

value : 0 1 3
```

For a perfect array:

```java
nums[i] == i
```

Before missing number:

```java
nums[i] == i
```

After missing number:

```java
nums[i] > i
```

We search for the first index where:

```java
nums[i] != i
```

---

## Dry Run

Input:

```java
[0,1,3]
```

---

Iteration 1

```java
left=0
right=2

mid=1
```

```java
nums[1]=1
```

Correct.

Move right.

```java
left=2
```

---

Iteration 2

```java
mid=2

nums[2]=3
```

Broken.

Move left.

```java
right=1
```

Stop.

Answer:

```java
2
```

---

## Code

```java
class Solution {

    public int missingNumber(int[] nums) {

        Arrays.sort(nums);

        int left = 0;
        int right = nums.length - 1;

        while(left <= right){

            int mid =
                left + (right-left)/2;

            if(nums[mid] == mid){

                left = mid + 1;
            }
            else{

                right = mid - 1;
            }
        }

        return left;
    }
}
```

---

## Complexity

```text
Time  : O(N log N)

Space : O(1)
```

# Problem 2

# LC540 - Single Element in Sorted Array

## Problem Description

Given:

```java
[1,1,2,3,3,4,4,8,8]
```

Every element appears twice except one.

Return:

```java
2
```

---

## Intuition

Before the single element:

Pairs start at EVEN index.

```java
1 1
3 3
4 4

0 1
2 3
4 5
```

After single element:

Pair positions shift.

```java
1 1 2 3 3 4 4

0 1
    3 4
      5 6
```

The first broken pair tells us where the answer lies.

---

## Dry Run

Input:

```java
[1,1,2,3,3,4,4,8,8]
```

---

Iteration 1

```java
mid=4
```

Even index.

Compare:

```java
nums[4]=3

nums[5]=4
```

Broken pair.

Search left.

---

Iteration 2

```java
mid=2
```

Compare:

```java
nums[2]=2

nums[3]=3
```

Broken.

Move left.

Eventually:

```java
2
```

found.

---

## Code

```java
class Solution {

    public int singleNonDuplicate(
            int[] nums) {

        int left = 0;
        int right = nums.length - 1;

        while(left < right){

            int mid =
                left + (right-left)/2;

            if(mid % 2 == 1)
                mid--;

            if(nums[mid]
                    ==
                    nums[mid+1]){

                left = mid + 2;
            }
            else{

                right = mid;
            }
        }

        return nums[left];
    }
}
```

---

## Complexity

```text
Time  : O(log N)

Space : O(1)
```

# Problem 3

# LC287 - Find Duplicate Number

## Problem Description

Given:

```java
[1,3,4,2,2]
```

Return:

```java
2
```

Only one number is duplicated.

---

## Intuition

Search range:

```java
1 → n
```

NOT indices.

Suppose:

```java
mid = 2
```

Count numbers:

```java
<= 2
```

Array:

```java
1,3,4,2,2
```

Count:

```java
1,2,2
```

Total:

```java
3
```

Expected:

```java
2
```

Too many numbers on left side.

Duplicate must lie there.

---

## Dry Run

Input:

```java
[1,3,4,2,2]
```

---

Iteration 1

```java
left=1
right=4

mid=2
```

Count:

```java
3
```

Since:

```java
3 > 2
```

Duplicate lies left.

```java
right=2
```

---

Iteration 2

```java
mid=1
```

Count:

```java
1
```

Valid.

Move right.

```java
left=2
```

Stop.

Answer:

```java
2
```

---

## Code

```java
class Solution {

    public int findDuplicate(
            int[] nums) {

        int left = 1;
        int right = nums.length - 1;

        while(left < right){

            int mid =
                left + (right-left)/2;

            int count = 0;

            for(int num : nums){

                if(num <= mid)
                    count++;
            }

            if(count > mid){

                right = mid;
            }
            else{

                left = mid + 1;
            }
        }

        return left;
    }
}
```

---

## Complexity

```text
Time  : O(N log N)

Space : O(1)
```
