# 02 - First / Last Occurrence Problems

## Variation Definition

In Classical Binary Search:

```text
Target Found
→ Return Index
```

In this variation:

```text
Target Found
→ Keep Searching
```

because we want:

```text
First Occurrence

Last Occurrence

Boundary Position
```

instead of any random occurrence.

---

# Why Does This Variation Exist?

Consider:

```java
nums = [5,7,7,8,8,10]

target = 8
```

Normal Binary Search:

```text
May return 3

May return 4
```

Both are correct for searching.

But if the problem asks:

```text
Find First Occurrence
```

Answer must be:

```java
3
```

If it asks:

```text
Find Last Occurrence
```

Answer must be:

```java
4
```

---

# Core Intuition

Normal Binary Search:

```java
if(nums[mid] == target)
    return mid;
```

Boundary Binary Search:

```java
if(nums[mid] == target)
```

Store answer.

Then continue searching.

Because:

```text
There may be another occurrence
further left or right.
```

---

# Recognition Signals

Think Boundary Search when you see:

* First Occurrence
* Last Occurrence
* Leftmost Position
* Rightmost Position
* First Valid
* Last Valid
* Boundary

Keywords:

```text
First

Last

Leftmost

Rightmost

Boundary
```

---

# Main Problem

## LC34 - Find First and Last Position of Element in Sorted Array

### Problem

Given:

```java
nums = [5,7,7,8,8,10]

target = 8
```

Return:

```java
[3,4]
```

If target not found:

```java
[-1,-1]
```

---

# Brute Force

Scan entire array.

Find:

```text
First occurrence

Last occurrence
```

Complexity:

```text
O(N)
```

---

# Optimal Intuition

We need:

```text
First Boundary
```

and

```text
Last Boundary
```

So run Binary Search twice.

---

# Part 1

## Find First Occurrence

Whenever:

```java
nums[mid] == target
```

Store:

```java
ans = mid
```

Then move:

```java
right = mid - 1
```

Why?

Because:

```text
There may be another target
further left.
```

---

# Dry Run

Input:

```java
[5,7,7,8,8,10]

target = 8
```

---

## Iteration 1

```java

nums[mid] = 7
```

Move right.

```java
left = 3
```

---

## Iteration 2

```java
mid = 4

nums[mid] = 8
```

Store:

```java
ans = 4
```

Move left.

```java

```

---

## Iteration 3

```java

nums[mid] = 8
```

Store:

```java
ans = 3
```

Loop ends.

First occurrence:

```java
3
```

---

# Code

```java
private int firstPosition(
        int[] nums,
        int target){

    int left = 0;
    int right = nums.length - 1;

    int ans = -1;

    while(left <= right){

        int mid =
            left + (right-left)/2;

        if(nums[mid] == target){

            ans = mid;

            right = mid - 1;
        }

        else if(nums[mid] < target){

            left = mid + 1;
        }

        else{

            right = mid - 1;
        }
    }

    return ans;
}
```

# Problem 2

## LC278 - First Bad Version

### Problem

Versions:

```text
1 2 3 4 5
```

Suppose:

```text
Version 4 is bad
```

Then:

```text
4 bad
5 bad
6 bad
...
```

Return:

```text
First Bad Version
```

---

# Key Observation

Versions form:

```text
Good Good Good Bad Bad Bad
```

or

```text
F F F F T T T T
```

This is a boundary problem.

We need:

```text
First True
```

---

# Intuition

When:

```java
isBadVersion(mid)
```

returns:

```text
true
```

Store answer.

Move left.

Because:

```text
There may be an earlier bad version.
```

---

# Code

```java
public class Solution
        extends VersionControl {

    public int firstBadVersion(int n) {

        int left = 1;
        int right = n;

        int ans = n;

        while(left <= right){

            int mid =
                left + (right-left)/2;

            if(isBadVersion(mid)){

                ans = mid;

                right = mid - 1;
            }

            else{

                left = mid + 1;
            }
        }

        return ans;
    }
}
```

---

# Complexity

```text
Time  : O(log N)

Space : O(1)
```

---

# Key Learning

Boundary Problems are not about:

```text
Finding Target
```

They are about:

```text
Finding Boundary
```

Examples:

```text
First Occurrence

Last Occurrence

First True

Last False

First Valid

Last Invalid
```

```text
Boundary Binary Search =
When condition becomes true,
```

# 03 - Lower Bound / Upper Bound Problems

## Variation Definition

In Classical Binary Search:

```text
Find exact target.
```

In Boundary Search:

```text
Find first or last occurrence.
```

In Lower Bound / Upper Bound:

```text
Find insertion positions.
```

We are not necessarily searching for a value.

We are searching for:

```text
Where should this value belong?
```

This variation is heavily used in:

* Searching
* Sorting
* Greedy
* LIS
* Binary Search on Answer

---

# Core Intuition

Consider:

```java
nums = [1,3,5,7,9]
target = 6
```

Target does not exist.

Question:

```text
Where should 6 be inserted?
```

Answer:

```text
Between 5 and 7
```

Index:

```java
3
```

This position is called:

```text
Lower Bound
```

---

# Lower Bound Definition

Lower Bound:

```text
First element
>= target
```

---

## Example

```java
nums = [1,3,5,7,9]
target = 6
```

First element:

```text
>= 6
```

is:

```text
7
```

Index:

```java
3
```

Answer:

```java
lowerBound = 3
```

---

# Upper Bound Definition

Upper Bound:

```text
First element
> target
```

---

## Example

```java
nums = [1,3,5,5,5,7,9]
target = 5
```

First element:

```text
> 5
```

is:

```text
7
```

Index:

```java
5
```

Answer:

```java
upperBound = 5
```

---

# Why Important?

Many interview problems are secretly:

```text
Find First >= X

Find First > X

Find First Valid Position
```

which are lower/upper bound problems.

---

# Recognition Signals

Think Lower Bound when you see:

* Insert Position
* First Greater
* First Greater Equal
* Ceiling
* Search Position

Think Upper Bound when you see:

* First Strictly Greater
* Next Larger Position

```java
2
```

```text

```

---

# Dry Run

```java
nums = [1,3,5,6]
target = 2
```

---

## Iteration 1

```java
left = 0
right = 3

mid = 1
```

```java
nums[mid] = 3
```

```java
3 >= 2
```

Possible answer.

```java
ans = 1
```

```java
right = 0
```

---

## Iteration 2

```java
mid = 0
nums[mid] = 1
```

```java

```

```java

```

---

# Code

```java
class Solution {

    public int searchInsert(
            int[] nums,
            int target) {

        int left = 0;
        int right = nums.length - 1;

        int ans = nums.length;

        while(left <= right){

            int mid =
                left + (right-left)/2;

            if(nums[mid] >= target){

                ans = mid;
                right = mid - 1;
            }
            else{

                left = mid + 1;
            }
        }

        return ans;
    }
}
```

```text
Time  : O(log N)

Space : O(1)
```

# Code

```java
class Solution {

    public char nextGreatestLetter(
            char[] letters,
            char target) {

        int left = 0;
        int right = letters.length - 1;

        int ans = 0;

        while(left <= right){

            int mid =
                left + (right-left)/2;

            if(letters[mid] > target){

                ans = mid;
                right = mid - 1;
            }
            else{

                left = mid + 1;
            }
        }

        return letters[ans];
    }
}
```

# Difference

| Type             | Condition       |
| ---------------- | --------------- |
| Lower Bound      | First >= target |
| Upper Bound      | First > target  |
| First Occurrence | First == target |
| Last Occurrence  | Last == target  |
