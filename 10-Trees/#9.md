# Variation 10 – Binary Search Tree (BST) Pattern

> **Pattern Goal:** Learn how to exploit the Binary Search Tree property to solve search, validation, insertion, deletion, and ordering problems efficiently.

---

# Core BST Property

Every node satisfies:

```text
Left Subtree < Root < Right Subtree
```

Example:

```text
          8
        /   \
       3     10
      / \      \
     1   6      14
        / \     /
       4   7   13
```

Unlike a normal binary tree, a BST allows us to eliminate half the tree after every comparison.

# Problem 1 — LC700 Search in BST

## Recognition

* Search
* Find
* Contains

## Core Intuition

Use BST comparisons to eliminate half the tree.

## Key Formula

```java
if (root == null || root.val == val)
    return root;

if (val < root.val)
    return searchBST(root.left, val);

return searchBST(root.right, val);
```

## Complexity

* Time: O(log n) average, O(n) worst
* Space: O(h)

## Interview Tip

Never search both subtrees in a BST.

## One-Line Revision

> Compare the target with the current node and recurse into only one subtree.

# Problem 2 — LC98 Validate BST

## Recognition

* Validate BST
* Check BST

## Core Intuition

A node is constrained by **all ancestors**, not just its parent.

Carry a valid range during DFS.

## Key Formula

```java
if (root.val <= min || root.val >= max)
    return false;

return validate(root.left, min, root.val)
    && validate(root.right, root.val, max);
```

## Complexity

* Time: O(n)
* Space: O(h)

## Interview Tip

Use `Long.MIN_VALUE` and `Long.MAX_VALUE`.

## One-Line Revision

> Carry the valid `(min, max)` range during DFS to ensure every node satisfies all ancestor constraints.

# Problem 3 — LC230 Kth Smallest Element in BST

## Recognition

* Kth Smallest
* Sorted Order
* Rank

## Core Intuition

Inorder traversal visits BST nodes in ascending order.

Maintain a counter while traversing.

## Key Formula

```java
inorder(root.left);

count++;

if (count == k)
    answer = root.val;

inorder(root.right);
```

## Complexity

* Time: O(h + k), worst O(n)
* Space: O(h)

## Interview Tip

Inorder is already sorted. No sorting is required.

## One-Line Revision

> Use inorder traversal and return the kth visited node.

# Problem 4 — LC701 Insert into BST

## Recognition

* Insert
* Add Node

## Core Intuition

Search until reaching a null position, then create the new node.

## Key Formula

```java
if (root == null)
    return new TreeNode(val);

if (val < root.val)
    root.left = insertIntoBST(root.left, val);
else
    root.right = insertIntoBST(root.right, val);

return root;
```

## Complexity

* Time: O(log n) average, O(n) worst
* Space: O(h)

## Interview Tip

Always reconnect the returned subtree using:

```java
root.left = ...
```

or

```java
root.right = ...
```

## One-Line Revision

> Search to the correct null position, insert the node, and return the updated subtree.

# Problem 5 — LC450 Delete Node in BST

## Recognition

* Delete
* Remove Node

## Core Intuition

Deletion has three cases:

1. Leaf → delete
2. One child → replace with child
3. Two children → replace with inorder successor, then delete the successor

## Find Inorder Successor

```java
private TreeNode findMin(TreeNode node) {
    while (node.left != null)
        node = node.left;
    return node;
}
```

## Key Formula

```java
if (root.left == null)
    return root.right;

if (root.right == null)
    return root.left;

TreeNode successor = findMin(root.right);

root.val = successor.val;

root.right = deleteNode(root.right, successor.val);
```

## Complexity

* Time: O(h)
* Space: O(h)

## Interview Tip

After copying the successor's value, don't forget to delete the original successor node.

## One-Line Revision

> Delete using three cases, replacing two-child nodes with their inorder successor.

# Problem 6 — LC530 Minimum Absolute Difference in BST

## Recognition

* Minimum Difference
* Closest Values
* BST

## Core Intuition

Inorder traversal produces a sorted sequence.

The minimum difference must occur between adjacent values.

## Key Formula

```java
if (prev != null)
    minDiff = Math.min(minDiff,
                       root.val - prev.val);

prev = root;
```

## Complexity

* Time: O(n)
* Space: O(h)

## Interview Tip

No need for `Math.abs()` because inorder guarantees:

```text
current >= previous
```

## One-Line Revision

> Compare each inorder node with the previously visited node to find the minimum adjacent difference.
