# Variation 4 — Global Answer DFS (Tree DP)

> **Goal:** Learn the Tree Dynamic Programming pattern where each recursive call **returns one value to its parent** while simultaneously **updating a global answer**.

---

# Pattern Recognition

Use this pattern when the interviewer says:

* Longest
* Maximum
* Diameter
* Best Path
* Maximum Sum
* Global Optimum

Mental Model

```text
Children return information

        ↓

Current node computes a candidate answer

        ↓

Update Global Answer

        ↓

Return information to parent
```

# Problem 1 — LC543 Diameter of Binary Tree

## Recognition Signals

* Diameter
* Longest Path
* Maximum Distance

Think

> Return Height + Update Diameter.

---

## Core Intuition

Every node computes

```text
Left Height + Right Height
```

This is the longest path passing through the current node.

Update the global diameter.

Return only the height to the parent.

---

## Formula

```text
Candidate Diameter

=

Left Height

+

Right Height
```

Return

```text
Height

=

max(left,right)+1
```

---

## Optimal Java

```java
class Solution {

    private int diameter = 0;

    public int diameterOfBinaryTree(TreeNode root) {

        height(root);

        return diameter;
    }

    private int height(TreeNode root) {

        if (root == null)
            return 0;

        int left = height(root.left);
        int right = height(root.right);

        diameter = Math.max(diameter,
                            left + right);

        return Math.max(left, right) + 1;
    }
}
```

---

## Complexity

```text
Time  : O(n)

Space : O(h)
```

---

## Interview Tip

Return **height**, not diameter.

The diameter is maintained globally.

---

## One-Line Revision

> Return height → Update diameter using `left + right`.

# Problem 2 — LC124 Binary Tree Maximum Path Sum

## Recognition Signals

* Maximum Sum
* Best Path
* Highest Path Sum

Think

> Return Best Downward Gain + Update Global Maximum.

---

## Core Intuition

Negative branches should never reduce the answer.

Ignore them using

```java
Math.max(0, subtreeGain)
```

Current node computes

```text
Left Gain

+

Node Value

+

Right Gain
```

Return only one branch to the parent.

---

## Formula

```java
left = Math.max(0, dfs(left));

right = Math.max(0, dfs(right));
```

Update

```java
left + right + root.val
```

Return

```java
root.val + Math.max(left,right)
```

---

## Optimal Java

```java
class Solution {

    private int maxSum = Integer.MIN_VALUE;

    public int maxPathSum(TreeNode root) {

        dfs(root);

        return maxSum;
    }

    private int dfs(TreeNode root) {

        if (root == null)
            return 0;

        int left = Math.max(0, dfs(root.left));

        int right = Math.max(0, dfs(root.right));

        maxSum = Math.max(maxSum,
                          left + right + root.val);

        return root.val +
               Math.max(left, right);
    }
}
```

---

## Complexity

```text
Time  : O(n)

Space : O(h)
```

---

## Interview Tip

Never include a negative branch.

A negative contribution always decreases the path sum.

---

## One-Line Revision

> Ignore negative gains → Update global path → Return best downward gain.

# Problem 3 — LC687 Longest Univalue Path

## Recognition Signals

* Same Value
* Equal Value
* Longest Equal Path

Think

> Return Longest Same-Value Chain.

---

## Core Intuition

A child contributes only if

```text
Child Value == Current Value
```

Otherwise,

the chain breaks.

Current node joins both matching chains to update the answer.

Return only the longer chain.

---

## Formula

If values match

```java
leftPath = left + 1;
```

Otherwise

```java
leftPath = 0;
```

Update

```java
leftPath + rightPath
```

Return

```java
Math.max(leftPath,rightPath)
```

---

## Optimal Java

```java
class Solution {

    private int longest = 0;

    public int longestUnivaluePath(TreeNode root) {

        dfs(root);

        return longest;
    }

    private int dfs(TreeNode root) {

        if (root == null)
            return 0;

        int left = dfs(root.left);
        int right = dfs(root.right);

        int leftPath = 0;
        int rightPath = 0;

        if (root.left != null &&
            root.left.val == root.val)
            leftPath = left + 1;

        if (root.right != null &&
            root.right.val == root.val)
            rightPath = right + 1;

        longest = Math.max(longest,
                           leftPath + rightPath);

        return Math.max(leftPath, rightPath);
    }
}
```

---

## Complexity

```text
Time  : O(n)

Space : O(h)
```

---

## Interview Tip

Only extend the chain when parent and child have the same value.

---

## One-Line Revision

> Extend matching chains → Update global answer → Return longest extendable chain.

# Variation Summary

Every recursive call performs **two independent jobs**.

### Job 1

Return information needed by the parent.

Examples

* Height
* Downward Gain
* Same-Value Chain

---

### Job 2

Update the global optimum.

Examples

* Diameter
* Maximum Path Sum
* Longest Univalue Path

The returned value is **not** the final answer.

The global variable stores the final answer.

---

# Variation Comparison

| Variation                       | Parent Needs              | Global Variable |
| ------------------------------- | ------------------------- | --------------- |
| Return Value DFS                | Height / Count            | ❌               |
| Root-to-Leaf DFS                | Carry State               | ❌               |
| Validation & Transformation DFS | Validation / Subtree Root | Sometimes       |
| Global Answer DFS               | Parent Information        | ✅ Always        |

# Golden Rules

### Rule 1

If the interviewer says

```text
Longest

Maximum

Best
```

Think

```text
Global Answer DFS
```

---

### Rule 2

Return only what the parent can extend.

---

### Rule 3

The complete path usually **ends at the current node**.

Use it to update the global answer.

Do not return it upward.

---

### Rule 4

The parent can continue through only **one child**.

Return the better branch.

# What You Learned

This variation introduces **Tree Dynamic Programming (Tree DP)**.

The key insight is:

* Every node computes a **local candidate answer**.
* The best candidate is stored in a **global variable**.
* The recursive function returns only the information required by the parent.

This pattern appears repeatedly in advanced tree interview problems.
