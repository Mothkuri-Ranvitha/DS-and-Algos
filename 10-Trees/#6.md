# Variation 6 – Tree Construction

> **Pattern Goal:** Learn how to reconstruct binary trees and BSTs using recursion.

---

# Recognition Signals

Whenever the interviewer says:

* Construct Binary Tree
* Rebuild Tree
* Recover Tree
* Build BST
* Traversal Arrays
* Sorted Array → BST
* BST from Preorder

Think immediately:

> **Choose the root → Divide the remaining data → Recursively build left and right subtrees.**

---

# Core Mental Model

Tree construction problems differ only in **how the root is chosen**.

| Input               | Root Selection                             |
| ------------------- | ------------------------------------------ |
| Preorder + Inorder  | First element of Preorder                  |
| Inorder + Postorder | Last element of Postorder                  |
| Sorted Array        | Middle element                             |
| BST Preorder        | Current preorder value within valid bounds |

Once the root is known:

1. Create the node.
2. Determine the left subtree.
3. Determine the right subtree.
4. Recursively repeat.

---

# Construction Pattern

```text
Choose Root

↓

Partition Remaining Data

↓

Build Left Subtree

↓

Build Right Subtree
```

# Problem 1 — LC105 Construct Binary Tree from Preorder & Inorder

## Recognition Signals

* Preorder + Inorder
* Construct Tree

## Core Intuition

* Preorder always gives the root first.
* Inorder tells where to split into left and right subtrees.
* Use a HashMap for O(1) inorder index lookup.

## Key Formula

```java
root = preorder[preStart];

index = map.get(root.val);

leftSize = index - inLeft;

left = build(preStart + 1);

right = build(preStart + leftSize + 1);
```

## Mental Model

```text
Root

↓

Split Inorder

↓

Left

↓

Right
```

## Time Complexity

```text
O(n)
```

## Space Complexity

```text
O(n)
```

(HashMap + recursion)

## Interview Tips

* Never search inorder linearly.
* Build a HashMap before recursion.
* The preorder index for the right subtree is computed using the left subtree size.

## One-Line Revision

> Pick the root from preorder, split using inorder, compute the left subtree size, then recursively build left and right.

# Problem 2 — LC106 Construct Binary Tree from Inorder & Postorder

## Recognition Signals

* Inorder + Postorder
* Construct Tree

## Core Intuition

* Postorder ends with the root.
* Inorder splits the tree.
* Since postorder visits **Left → Right → Root**, after removing the root, the next available node belongs to the **right subtree**, so build the right subtree first.

## Key Formula

```java
root = postorder[postEnd];

index = map.get(root.val);

rightSize = inRight - index;

right = build(postEnd - 1);

left = build(postEnd - rightSize - 1);
```

## Mental Model

```text
Root

↓

Split Inorder

↓

Right

↓

Left
```

## Time Complexity

```text
O(n)
```

## Space Complexity

```text
O(n)
```

## Interview Tips

* Build the right subtree before the left subtree.
* `rightSize` tells how many nodes belong to the right subtree.

## One-Line Revision

> Take the root from the end of postorder, split using inorder, build the right subtree first, then the left.

# Problem 3 — LC108 Convert Sorted Array to BST

## Recognition Signals

* Sorted Array
* Height Balanced BST

## Core Intuition

The middle element naturally divides the array into two balanced halves.

## Key Formula

```java
mid = left + (right - left) / 2;

root = nums[mid];

left = build(left, mid - 1);

right = build(mid + 1, right);
```

## Mental Model

```text
Middle

↓

Root

↓

Left Half

↓

Right Half
```

## Time Complexity

```text
O(n)
```

## Space Complexity

```text
O(log n)
```

(Balanced recursion stack)

## Interview Tips

Always use

```java
left + (right - left) / 2
```

instead of

```java
(left + right) / 2
```

to avoid integer overflow.

## One-Line Revision

> Choose the middle element as the root and recursively build the left and right halves.

# Problem 4 — LC1008 Construct BST from Preorder

## Recognition Signals

* BST
* Preorder
* Construct BST

## Core Intuition

Instead of splitting arrays, use BST bounds.

Each recursive call checks whether the current preorder value belongs to the current subtree.

## Key Formula

```java
if(value < low || value > high)
    return null;

root = preorder[index++];

left = build(low, root.val);

right = build(root.val, high);
```

## Mental Model

```text
Check Bounds

↓

Create Root

↓

Move Pointer

↓

Left

↓

Right
```

## Time Complexity

```text
O(n)
```

## Space Complexity

```text
O(h)
```

where `h` is the height of the BST.

## Interview Tips

Do **not** increment the preorder pointer if the current value violates the bounds. That value belongs to an ancestor's subtree.

## One-Line Revision

> Use a global preorder pointer and valid BST bounds to construct the tree in O(n).

# Interview Summary

Tree construction questions almost always reduce to four steps:

1. Identify the root.
2. Partition the remaining input.
3. Recursively build the left subtree.
4. Recursively build the right subtree.

The only thing that changes between problems is **how the root is identified and how the remaining input is partitioned**.
