* Serialize / Deserialize Tree

# Core Mental Model

Unlike traversal problems where we return an answer, these problems require us to **transform or represent the tree**.

```text
Tree
│
├── Serialize → String
├── Deserialize → Tree
├── Flatten → Linked List
└── Iterator → Lazy Inorder Traversal
```

The tree structure is just as important as the node values.

# Problem 1 — LC297 Serialize and Deserialize Binary Tree

## Recognition Signals

* Serialize Tree
* Deserialize Tree
* Encode / Decode Tree

## Core Intuition

* Preorder traversal naturally supports reconstruction.
* Store **null nodes** using a special marker (e.g., `#`) so the tree structure is preserved.
* During deserialization, read the serialized data in preorder and recursively rebuild the tree.

## Mental Model

```text
Serialize

Node
↓
Left
↓
Right

----------------

Deserialize

Read Value
↓
Create Node
↓
Left
↓
Right
```

## Key Formula

```java
Serialize:
Node → Left → Right

Null → "#"

Deserialize:
Read

↓

If "#"
Return null

↓

Create Node

↓

Left

↓

Right
```

## Time Complexity

```
O(n)
```

## Space Complexity

```
O(n)
```

(Queue + recursion stack)

## Interview Tips

* Without null markers, different trees can produce the same preorder traversal.
* Serialization and deserialization **must follow the same traversal order**.

## One-Line Revision

> Serialize using preorder with null markers and deserialize by recursively reading the same preorder sequence.

# Problem 2 — LC114 Flatten Binary Tree to Linked List

## Recognition Signals

* Flatten Tree
* Convert Tree to Linked List
* In-place Modification

## Core Intuition

Two approaches:

### Approach 1 (Simple)

* Flatten left subtree.
* Flatten right subtree.
* Move the left subtree to the right.
* Attach the original right subtree to the end.

Time: **O(n²)**

### Approach 2 (Optimal)

Traverse in **Reverse Preorder (Right → Left → Root)** while maintaining a global `prev` pointer.

Each node connects directly to the previously processed node.

## Mental Model

```text
Reverse Preorder

Right
↓
Left
↓
Root

↓

root.right = prev

↓

prev = root
```

## Key Formula

```java
flatten(root.right);

flatten(root.left);

root.right = prev;

root.left = null;

prev = root;
```

## Time Complexity

```
O(n)
```

## Space Complexity

```
O(h)
```

where `h` is the tree height.

## Interview Tips

* Mention the O(n²) approach first.
* Then optimize using reverse preorder and a global `prev` pointer.

## One-Line Revision

> Reverse preorder with a previous pointer rewires the tree into a preorder linked list in O(n).

# Problem 3 — LC173 Binary Search Tree Iterator

## Recognition Signals

* BST Iterator
* Next Smallest Element
* Lazy Traversal
* Streaming Tree

## Core Intuition

* BST inorder traversal produces sorted order.
* Instead of storing the full inorder traversal, maintain only the path to the next smallest node using a stack.
* Push the entire left chain initially.
* After visiting a node, push the left chain of its right subtree.

## Mental Model

```text
Push Left Chain

↓

Pop Smallest

↓

Push Left Chain
of Right Child
```

## Key Formula

```java
pushLeft(root);

node = stack.pop();

pushLeft(node.right);

return node.val;
```

## Time Complexity

Constructor

```
O(h)
```

next()

```
Amortized O(1)
```

hasNext()

```
O(1)
```

## Space Complexity

```
O(h)
```

## Interview Tips

* Do **not** build the entire inorder array.
* This is **lazy inorder traversal**.
* Every node is pushed once and popped once, giving amortized O(1) for `next()`.

## One-Line Revision

> Maintain a stack of the current left path to lazily perform inorder traversal one node at a time.

# Interview Summary

Tree serialization and modification problems focus on preserving or transforming tree structure rather than computing values.

Most solutions follow this pattern:

1. Traverse the tree.
2. Maintain additional state (queue, stack, or pointer).
3. Modify or reconstruct the tree while preserving the required order.
