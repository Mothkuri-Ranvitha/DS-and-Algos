# 12-Tree

# Variation 2 — Root-to-Leaf DFS (Top-Down Traversal)

> **Goal:** Learn how to solve path-based tree problems by **carrying information from the root to the current node**, instead of collecting answers from children.

---

# Pattern Recognition

Use this pattern when the question asks:

* Root → Leaf Path
* Path Sum
* Current Path
* Print All Paths
* Store Paths
* Path String

Mental Model

```text
Root
  │
  ▼
Carry Information
  │
  ▼
Current Node
  │
  ▼
Leaf
```

Instead of asking:

> **"What should my child return?"**

Ask:

> **"What information should I carry to my child?"**

---

# Generic Template

```java
void dfs(TreeNode root, State state) {

    if (root == null)
        return;

    // Update state

    dfs(root.left, updatedState);

    dfs(root.right, updatedState);
}
```

The recursive call usually returns **void** because the important information is carried **downward**.

---

# Problem 1 — LC112 Path Sum

## Recognition Signals

* Root to Leaf
* Path Sum
* Target Sum
* Exists Path

Think

> Carry the **remaining target**.

---

## Core Intuition

Instead of calculating the current sum,

keep reducing the target.

Example

```text
Target = 22

Visit 5

Remaining = 17

Visit 4

Remaining = 13

Visit 11

Remaining = 2

Visit 2

Remaining = 0
```

Reached a leaf.

Valid path found.

---

## Important Rule

Never check the answer before reaching a **leaf**.

A partial path is **not** a valid Root → Leaf path.

---

## Formula

```text
Remaining Target

=

Remaining Target

-

Current Node Value
```

---

## Optimal Java

```java
class Solution {

    public boolean hasPathSum(TreeNode root, int targetSum) {

        if (root == null)
            return false;

        if (root.left == null &&
            root.right == null)
            return targetSum == root.val;

        return hasPathSum(root.left,
                          targetSum - root.val)
            ||
               hasPathSum(root.right,
                          targetSum - root.val);
    }
}
```

---

## Complexity

```text
Time  : O(n)

Space : O(h)
```

---

## Interview Tip

Use **remaining target** instead of **current sum**.

Cleaner recursion.

---

## One-Line Revision

> Carry remaining target → Check only at leaf.

# Problem 2 — LC113 Path Sum II

## Recognition Signals

* Return all paths
* Store paths
* Print all Root → Leaf paths

Think

> Carry the path + Backtrack.

---

## Core Intuition

Carry

* Remaining Target
* Current Path

Whenever a valid leaf is found,

store a **copy** of the current path.

---

## The Three Steps of Backtracking

```java
path.add(root.val);      // Choose

dfs(...);                // Explore

path.remove(path.size()-1); // Undo
```

Every add must have exactly one remove.

---

## Why Copy the Path?

Wrong

```java
result.add(path);
```

Correct

```java
result.add(new ArrayList<>(path));
```

Otherwise every stored answer changes during backtracking.

---

## Optimal Java

```java
class Solution {

    public List<List<Integer>> pathSum(TreeNode root,
                                       int targetSum) {

        List<List<Integer>> result =
                new ArrayList<>();

        List<Integer> path =
                new ArrayList<>();

        dfs(root, targetSum, path, result);

        return result;
    }

    private void dfs(TreeNode root,
                     int target,
                     List<Integer> path,
                     List<List<Integer>> result) {

        if (root == null)
            return;

        path.add(root.val);

        if (root.left == null &&
            root.right == null &&
            target == root.val) {

            result.add(new ArrayList<>(path));
        }

        dfs(root.left,
            target - root.val,
            path,
            result);

        dfs(root.right,
            target - root.val,
            path,
            result);

        path.remove(path.size() - 1);
    }
}
```

---

## Complexity

```text
Time  : O(n × h)

Space : O(h)
```

(`h` = tree height)

---

## Interview Tip

Whenever you hear

* All solutions
* Generate all paths
* Return every answer

Think

```text
Choose

↓

Explore

↓

Undo
```

---

## One-Line Revision

> Carry path → Store copy → Backtrack.

# Problem 3 — LC257 Binary Tree Paths

## Recognition Signals

* Return path strings
* Print paths
* Root → Leaf paths

Think

> Carry a String.

---

## Core Intuition

Instead of

```text
[1,2,5]
```

build

```text
"1->2->5"
```

Traversal remains exactly the same.

Only the carried state changes.

---

## Why No Backtracking?

Strings are **immutable**.

Every recursive call receives a new string.

So there is nothing to undo.

---

## Optimal Java

```java
class Solution {

    public List<String> binaryTreePaths(TreeNode root) {

        List<String> result = new ArrayList<>();

        dfs(root, "", result);

        return result;
    }

    private void dfs(TreeNode root,
                     String path,
                     List<String> result) {

        if (root == null)
            return;

        if (path.isEmpty())
            path = String.valueOf(root.val);
        else
            path = path + "->" + root.val;

        if (root.left == null &&
            root.right == null) {

            result.add(path);
            return;
        }

        dfs(root.left, path, result);
        dfs(root.right, path, result);
    }
}
```

---

## Complexity

```text
Time  : O(n × h)

Space : O(h)
```

---

## Interview Tip

Backtracking is required only for **mutable objects** like `List` or `StringBuilder`.

Not for `String`.

---

## One-Line Revision

> Carry string → Store at leaf → No backtracking.

# Variation 1 vs Variation 2

| Variation 1               | Variation 2                   |
| ------------------------- | ----------------------------- |
| Bottom-Up DFS             | Top-Down DFS                  |
| Children return answers   | Parent carries state          |
| `int/boolean dfs(node)`   | `dfs(node, state)`            |
| Height, Count, Validation | Path Sum, Paths, Root-to-Leaf |

# Golden Rules

### Rule 1

If the problem says

```text
Root → Leaf
```

Think

```text
Carry information downward.
```

---

### Rule 2

Check the answer **only at the leaf**.

---

### Rule 3

If using a mutable object (`List`, `StringBuilder`),

always

```text
Choose

↓

Explore

↓

Undo
```

---

### Rule 4

If using immutable objects (`int`, `String`),

backtracking is not required.

---

# What You Learned

Root-to-Leaf DFS is the **Top-Down Tree Pattern**.

The recursive calls don't compute answers for the parent.

Instead, they **carry state** from the root toward the leaves.

Only the carried state changes:

* Integer → Path Sum
* List → Path Sum II
* String → Binary Tree Paths

The traversal remains exactly the same.

Once you recognize **"Root → Leaf"**, you should immediately think:

> **Carry information downward and validate when you reach a leaf.**
