# Problem 1 — LC236 Lowest Common Ancestor of a Binary Tree

## Recognition Signals

* Lowest Common Ancestor
* Common Parent
* Common Node
* Two Target Nodes

Think

> Search Left + Search Right + Propagate Upward.

---

## Core Intuition

Every recursive call returns one of four things:

* `null`
* `p`
* `q`
* LCA

If

* Left subtree finds one target
* Right subtree finds the other

Current node is the Lowest Common Ancestor.

---

## Decision Table

| Left | Right | Return     |
| ---- | ----- | ---------- |
| null | null  | null       |
| p/q  | null  | p/q        |
| null | p/q   | p/q        |
| p    | q     | root (LCA) |

---

## Optimal Java

```java
class Solution {

    public TreeNode lowestCommonAncestor(TreeNode root,
                                         TreeNode p,
                                         TreeNode q) {

        if (root == null)
            return null;

        if (root == p || root == q)
            return root;

        TreeNode left =
            lowestCommonAncestor(root.left, p, q);

        TreeNode right =
            lowestCommonAncestor(root.right, p, q);

        if (left != null && right != null)
            return root;

        return left != null ? left : right;
    }
}
```

---

## Complexity

```
Time  : O(n)

Space : O(h)
```

---

## Interview Tip

The first node that receives one target from each subtree is the Lowest Common Ancestor.

---

## One-Line Revision

> Search both subtrees. If both return non-null, current node is the LCA; otherwise return the non-null result.

# Problem 2 — LC235 Lowest Common Ancestor of a BST

## Recognition Signals

* BST
* Lowest Common Ancestor
* Search Tree

Think

> Use BST Property.

---

## Core Intuition

BST Property:

```text
Left < Root < Right
```

Three cases:

* Both targets smaller → Go Left
* Both targets larger → Go Right
* Split occurs → Current node is LCA

---

## Recursive Solution

```java
class Solution {

    public TreeNode lowestCommonAncestor(TreeNode root,
                                         TreeNode p,
                                         TreeNode q) {

        if (p.val < root.val &&
            q.val < root.val)
            return lowestCommonAncestor(root.left, p, q);

        if (p.val > root.val &&
            q.val > root.val)
            return lowestCommonAncestor(root.right, p, q);

        return root;
    }
}
```

---

## Iterative Solution (Preferred)

```java
class Solution {

    public TreeNode lowestCommonAncestor(TreeNode root,
                                         TreeNode p,
                                         TreeNode q) {

        while (root != null) {

            if (p.val < root.val &&
                q.val < root.val)
                root = root.left;

            else if (p.val > root.val &&
                     q.val > root.val)
                root = root.right;

            else
                return root;
        }

        return null;
    }
}
```

---

## Complexity

Recursive

```
Time  : O(h)

Space : O(h)
```

Iterative

```
Time  : O(h)

Space : O(1)
```

---

## Interview Tip

Unlike a normal Binary Tree, a BST lets us eliminate half the search space at every step.

---

## One-Line Revision

> If both targets are on the same side, move there; otherwise current node is the LCA.

# Problem 3 — LC863 All Nodes Distance K in Binary Tree

## Recognition Signals

* Distance K
* Nodes at Distance
* Move Up and Down
* Shortest Distance

Think

> Parent Map + BFS.

---

## Core Intuition

A binary tree allows movement only downward.

Build a parent map to enable upward movement, turning the tree into an undirected graph.

Then run BFS from the target node.

---

## Algorithm

### Phase 1

Build Parent Map

```text
Child → Parent
```

---

### Phase 2

Run BFS

Explore:

* Left
* Right
* Parent

Stop when distance equals `k`.

---

## Optimal Java

```java
class Solution {

    private Map<TreeNode, TreeNode> parent = new HashMap<>();

    public List<Integer> distanceK(TreeNode root,
                                   TreeNode target,
                                   int k) {

        buildParent(root, null);

        Queue<TreeNode> queue = new LinkedList<>();
        Set<TreeNode> visited = new HashSet<>();

        queue.offer(target);
        visited.add(target);

        int distance = 0;

        while (!queue.isEmpty()) {

            if (distance == k)
                break;

            int size = queue.size();

            for (int i = 0; i < size; i++) {

                TreeNode node = queue.poll();

                if (node.left != null &&
                    visited.add(node.left))
                    queue.offer(node.left);

                if (node.right != null &&
                    visited.add(node.right))
                    queue.offer(node.right);

                TreeNode par = parent.get(node);

                if (par != null &&
                    visited.add(par))
                    queue.offer(par);
            }

            distance++;
        }

        List<Integer> ans = new ArrayList<>();

        while (!queue.isEmpty())
            ans.add(queue.poll().val);

        return ans;
    }

    private void buildParent(TreeNode node,
                             TreeNode par) {

        if (node == null)
            return;

        parent.put(node, par);

        buildParent(node.left, node);

        buildParent(node.right, node);
    }
}
```

---

## Complexity

```
Time  : O(n)

Space : O(n)
```

---

## Interview Tip

After adding parent pointers, the tree behaves like an undirected graph. A visited set is required to prevent revisiting nodes.

---

## One-Line Revision

> Build parent pointers, then perform BFS from the target to find all nodes exactly `k` edges away.

# Pattern Summary

This variation introduces **Ancestor Search**.

The recursive call no longer returns heights or sums.

Instead, it returns:

* Target node
* Lowest Common Ancestor
* Search result

The parent combines these returned values to determine the final answer.
