# Variation 11 – Tree Dynamic Programming (Tree DP)

> **Pattern Goal:** Learn how to solve optimization problems on trees by returning information from child nodes and combining it at the parent.

---

# What is Tree DP?

Tree DP means:

> **Each recursive call returns information about its subtree, and the parent combines those results to solve a larger subproblem.**

Unlike normal DFS, where recursion is only used to visit nodes, Tree DP uses recursion to **compute and propagate states upward**.

---

# Recognition Signals

Whenever you hear:

* Maximum Path
* Longest Path
* Best Answer
* Parent depends on children
* Pick or Skip
* Alternate directions
* Coverage / Monitoring
* Return multiple values

Think immediately:

> **Tree DP**

---

# Generic Tree DP Template

```text
DFS(node)

↓

Solve Left

↓

Solve Right

↓

Combine Child Results

↓

Update Global Answer (if needed)

↓

Return State to Parent
```

---

# Mental Model

Every child reports information to its parent.

```text
            Parent
           /      \
          /        \
     Left Info   Right Info

Parent combines them.

↓

Returns new information upward.
```

The entire tree is solved **bottom-up**.

# Problem 1 — LC124 Binary Tree Maximum Path Sum ⭐⭐⭐⭐⭐

## Pattern

Return **one value** while maintaining a **global answer**.

### Return

```text
Maximum gain this node can contribute to its parent.
```

### Global Answer

```text
leftGain + root + rightGain
```

### Return Formula

```java
return root.val + Math.max(leftGain, rightGain);
```

### Global Formula

```java
answer = Math.max(answer,
                  leftGain + root.val + rightGain);
```

### Key Insight

* Parent can extend only one branch.
* Local answer may use both branches.

### Complexity

* Time: O(n)
* Space: O(h)

### One-Line Revision

> Return the best single-branch gain while updating the maximum path passing through each node.

# Problem 2 — LC337 House Robber III ⭐⭐⭐⭐⭐

## Pattern

Return **two DP states**.

### Return

```text
[rob, skip]
```

Meaning:

```text
rob

↓

Maximum money if current node IS robbed.
```

```text
skip

↓

Maximum money if current node is NOT robbed.
```

### DP Formula

```java
rob =
root.val +
left.skip +
right.skip;
```

```java
skip =
max(left.rob,left.skip)
+
max(right.rob,right.skip);
```

### Return

```java
return new int[]{rob, skip};
```

### Final Answer

```java
Math.max(ans[0], ans[1]);
```

### Complexity

* Time: O(n)
* Space: O(h)

### One-Line Revision

> Return both rob and skip states so the parent can choose according to the robbery rule.

# Problem 3 — LC1372 Longest ZigZag Path ⭐⭐⭐⭐

## Pattern

Return **direction-based states**.

### Return

```text
[goLeft, goRight]
```

Meaning:

* goLeft → Longest ZigZag if first move is left.
* goRight → Longest ZigZag if first move is right.

### DP Formula

```java
goLeft = left[1] + 1;

goRight = right[0] + 1;
```

### Global Answer

```java
answer =
Math.max(answer,
         Math.max(goLeft, goRight));
```

### Base Case

```java
return new int[]{-1,-1};
```

### Complexity

* Time: O(n)
* Space: O(h)

### One-Line Revision

> Return two direction states and always extend using the opposite direction from the child.

# Problem 4 — LC968 Binary Tree Cameras ⭐⭐⭐⭐⭐

## Pattern

Return a **state machine**.

### States

```text
0 → NEED

1 → CAMERA

2 → COVERED
```

### Rules

#### Child Needs Camera

```java
camera++;

return CAMERA;
```

---

#### Child Has Camera

```java
return COVERED;
```

---

#### Otherwise

```java
return NEED;
```

### Root Check

```java
if (dfs(root) == NEED)
    camera++;
```

### Complexity

* Time: O(n)
* Space: O(h)

### One-Line Revision

> Return coverage states upward and place cameras only when a child explicitly needs one.

# Problem 5 — LC687 Longest Univalue Path ⭐⭐⭐⭐

## Pattern

Return **one value** with a **condition check**.

### Return

```text
Longest same-value branch.
```

### DP Formula

```java
if (root.left != null &&
    root.left.val == root.val)
    leftPath = left + 1;

if (root.right != null &&
    root.right.val == root.val)
    rightPath = right + 1;
```

### Global Answer

```java
answer =
Math.max(answer,
         leftPath + rightPath);
```

### Return

```java
return Math.max(leftPath,
                rightPath);
```

### Complexity

* Time: O(n)
* Space: O(h)

### One-Line Revision

> Extend only matching-value branches, update the global answer using both branches, and return the longer branch upward.

# One-Line Pattern Revision

> **Tree DP solves optimization problems by returning the exact information a parent needs from each subtree, combining child states locally, optionally updating a global answer, and propagating only the required state upward.**
