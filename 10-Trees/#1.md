# Pattern Recognition

Use this pattern when the question asks:

* Height
* Depth
* Count
* Sum
* Compare Trees
* Validate
* Symmetric
* Minimum / Maximum

Mental Model

```
Current Node

↓

Ask Left Subtree

↓

Ask Right Subtree

↓

Combine Answers

↓

Return Answer
```

---

# Generic Recursive Formula

```java
ReturnType dfs(TreeNode root){

    if(root == null)
        return BaseValue;

    LeftAnswer = dfs(root.left);

    RightAnswer = dfs(root.right);

    return Combine(left, right, root);
}
```

Everything changes only in:

* Return Type
* Base Case
* Combine Logic

# Problem 1 — LC104 Maximum Depth of Binary Tree

## Recognition Signals

* Height
* Depth
* Longest root-to-leaf path

Think

> Every node returns the **height of its subtree**.

---

## Core Intuition

A node cannot know its own height until both children tell their heights.

Information flows

```
Leaves

↓

Parents

↓

Root
```

---

## Formula

```
height(node)

=

max(leftHeight, rightHeight)

+

1
```

---

## Dry Run

```
        1
      /   \
     2     3
    /
   4
```

```
4 → 1

2 → 2

3 → 1

1 → 3
```

---

## Optimal Java

```java
class Solution {

    public int maxDepth(TreeNode root) {

        if (root == null)
            return 0;

        int left = maxDepth(root.left);

        int right = maxDepth(root.right);

        return Math.max(left, right) + 1;
    }
}
```

---

## Complexity

```
Time  : O(n)

Space : O(h)
```

---

## Interview Tip

Height problems almost always use **Postorder DFS**.

---

## One-Line Revision

> Return subtree height → `max(left, right) + 1`

# Problem 2 — LC100 Same Tree

## Recognition Signals

* Same Tree
* Identical Tree
* Equal Trees

Think

> Every recursive call returns **true or false**.

---

## Core Intuition

Never compare the whole trees.

Compare only one pair of nodes.

Then ask

* Are left subtrees same?
* Are right subtrees same?

---

## Base Cases

```
Both null

↓

true
```

```
One null

↓

false
```

```
Different values

↓

false
```

---

## Formula

```
Current Values Same

AND

Left Trees Same

AND

Right Trees Same
```

---

## Optimal Java

```java
class Solution {

    public boolean isSameTree(TreeNode p, TreeNode q) {

        if (p == null && q == null)
            return true;

        if (p == null || q == null)
            return false;

        if (p.val != q.val)
            return false;

        return isSameTree(p.left, q.left)
            && isSameTree(p.right, q.right);
    }
}
```

---

## Complexity

```
Time  : O(n)

Space : O(h)
```

---

## Interview Tip

Questions asking

* Same?
* Equal?
* Valid?

usually return **boolean**.

---

## One-Line Revision

> Compare current → compare left → compare right.

# Problem 3 — LC101 Symmetric Tree

## Recognition Signals

* Symmetric Tree
* Mirror Tree
* Reflection

Think

> Boolean DFS + Mirror Comparison.

---

## Core Intuition

Don't compare

```
Left.left

↓

Right.left
```

Compare

```
Left.left

↓

Right.right
```

and

```
Left.right

↓

Right.left
```

Mirror recursion.

---

## Formula

```
Current Values Same

AND

Outside Children Mirror

AND

Inside Children Mirror
```

Outside

```
left.left

↓

right.right
```

Inside

```
left.right

↓

right.left
```

---

## Optimal Java

```java
class Solution {

    public boolean isSymmetric(TreeNode root) {

        if (root == null)
            return true;

        return mirror(root.left, root.right);
    }

    private boolean mirror(TreeNode left, TreeNode right) {

        if (left == null && right == null)
            return true;

        if (left == null || right == null)
            return false;

        if (left.val != right.val)
            return false;

        return mirror(left.left, right.right)
            && mirror(left.right, right.left);
    }
}
```

---

## Complexity

```
Time  : O(n)

Space : O(h)
```

---

## Interview Tip

Mirror problems always compare

```
Outside

+

Inside
```

instead of left-left and right-right.

---

## One-Line Revision

> Same Tree + Cross Comparison.

# Problem 4 — LC111 Minimum Depth of Binary Tree

## Recognition Signals

* Minimum Depth
* Closest Leaf
* Shortest Root-to-Leaf

Think

> Return minimum subtree depth.

---

## Biggest Interview Trap ⭐⭐⭐⭐⭐

Never write

```java
Math.min(left, right) + 1
```

without checking whether both children exist.

---

## Why?

```
      1
     /
    2
```

Left

```
1
```

Right

```
0
```

```
min(1,0)+1

=

1

❌ Wrong
```

A missing child is **not** a valid path.

---

## Cases

Leaf

```
return 1
```

Only Left

```
left + 1
```

Only Right

```
right + 1
```

Both

```
min(left, right) + 1
```

---

## Optimal Java

```java
class Solution {

    public int minDepth(TreeNode root) {

        if (root == null)
            return 0;

        if (root.left == null && root.right == null)
            return 1;

        if (root.left == null)
            return minDepth(root.right) + 1;

        if (root.right == null)
            return minDepth(root.left) + 1;

        return Math.min(minDepth(root.left),
                        minDepth(root.right)) + 1;
    }
}
```

---

## Complexity

```
Time  : O(n)

Space : O(h)
```

---

## Interview Tip

Minimum depth is measured to the nearest **leaf**, not the nearest null.

---

## One-Line Revision

> Ignore missing child. Choose minimum only among existing paths.

# Problem 5 — LC222 Count Complete Tree Nodes

## Recognition Signals

* Count Nodes
* Complete Binary Tree

Think

> Return number of nodes in the subtree.

---

## Core Intuition

Each node returns

```
Nodes in Left Subtree

+

Nodes in Right Subtree

+

Current Node
```

---

## Formula

```
count(node)

=

left

+

right

+

1
```

---

## Simple DFS Solution

```java
class Solution {

    public int countNodes(TreeNode root) {

        if (root == null)
            return 0;

        return 1
             + countNodes(root.left)
             + countNodes(root.right);
    }
}
```

---

## Optimization

Since the tree is **Complete**:

If

```
Left Height

==

Right Height
```

then the subtree is **Perfect**.

Nodes

```
2^height - 1
```

Otherwise recurse normally.

---

## Optimized Java

```java
class Solution {

    public int countNodes(TreeNode root) {

        if (root == null)
            return 0;

        int leftHeight = leftHeight(root);
        int rightHeight = rightHeight(root);

        if (leftHeight == rightHeight)
            return (1 << leftHeight) - 1;

        return 1
             + countNodes(root.left)
             + countNodes(root.right);
    }

    private int leftHeight(TreeNode node) {

        int h = 0;

        while (node != null) {
            h++;
            node = node.left;
        }

        return h;
    }

    private int rightHeight(TreeNode node) {

        int h = 0;

        while (node != null) {
            h++;
            node = node.right;
        }

        return h;
    }
}
```

---

## Complexity

Simple DFS

```
Time  : O(n)

Space : O(h)
```

Optimized

```
Time  : O((log n)²)

Space : O(log n)
```

---

## Interview Tip

Always explain the DFS solution first.

Then optimize using the **Complete Binary Tree** property.

---

## One-Line Revision

> Count = Left + Right + 1.

# What You Learned

Every recursive tree problem follows the same framework:

1. **What should each recursive call return?**
2. **What is the base case?**
3. **How do I combine the left and right answers?**

Only the **return type** and the **combine logic** change.

Master this framework, and you'll recognize the solution pattern for a large class of Binary Tree interview problems.
