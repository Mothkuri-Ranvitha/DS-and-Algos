# Variation 3 — Tree Validation & Transformation DFS

> **Goal:** Learn how to validate or transform a tree using DFS. Every recursive call either returns information needed by its parent or returns the transformed subtree.

---

# Pattern Recognition

Use this pattern when the interviewer says:

* Balanced Tree
* Validate Tree
* Validate BST
* Mirror Tree
* Invert Tree
* Merge Tree
* Transform Tree

Mental Model

```text
Children solve their subtrees

        ↓

Parent uses their result

        ↓

Return information / transformed subtree
```

---

# Generic Template

```java
ReturnType dfs(TreeNode root) {

    if (root == null)
        return Base;

    ReturnType left = dfs(root.left);
    ReturnType right = dfs(root.right);

    // Validate or Transform

    return answer;
}
```

# Problem 1 — LC110 Balanced Binary Tree

## Recognition Signals

* Balanced
* Height Difference
* Height of Left/Right Subtree

Think

> Return Height + Early Termination.

---

## Core Intuition

Every node needs only:

* Left Height
* Right Height

If

```text
abs(left - right) > 1
```

the subtree is not balanced.

Instead of computing heights multiple times, return a special value:

```text
-1
```

to indicate an unbalanced subtree.

This combines validation and height calculation into one DFS.

---

## Recursive Formula

```text
Get Left Height

↓

Get Right Height

↓

If either is -1

↓

Return -1

↓

If difference > 1

↓

Return -1

↓

Else

Return max(left, right) + 1
```

---

## Optimal Java

```java
class Solution {

    public boolean isBalanced(TreeNode root) {
        return height(root) != -1;
    }

    private int height(TreeNode root) {

        if (root == null)
            return 0;

        int left = height(root.left);

        if (left == -1)
            return -1;

        int right = height(root.right);

        if (right == -1)
            return -1;

        if (Math.abs(left - right) > 1)
            return -1;

        return Math.max(left, right) + 1;
    }
}
```

---

## Complexity

```text
Time  : O(n)

Space : O(h)
```

---

## Interview Tip

Use a **sentinel value (`-1`)** to combine height calculation and balance validation.

---

## One-Line Revision

> Return height; use `-1` to indicate an unbalanced subtree.

# Problem 2 — LC98 Validate Binary Search Tree

## Recognition Signals

* Validate BST
* BST Rules
* Entire Left/Right Subtree

Think

> Carry Valid Range.

---

## Core Intuition

A node must satisfy constraints from **all its ancestors**, not just its parent.

Every recursive call carries:

```text
(minAllowed, maxAllowed)
```

Check

```text
min < node.val < max
```

Then update the range for children.

---

## Recursive Formula

Left Child

```text
(min, node.val)
```

Right Child

```text
(node.val, max)
```

---

## Optimal Java

```java
class Solution {

    public boolean isValidBST(TreeNode root) {
        return dfs(root, Long.MIN_VALUE, Long.MAX_VALUE);
    }

    private boolean dfs(TreeNode root,
                        long min,
                        long max) {

        if (root == null)
            return true;

        if (root.val <= min || root.val >= max)
            return false;

        return dfs(root.left, min, root.val)
            &&
               dfs(root.right, root.val, max);
    }
}
```

---

## Complexity

```text
Time  : O(n)

Space : O(h)
```

---

## Interview Tip

Never compare only parent and child.

Every node must satisfy the range imposed by all ancestors.

---

## One-Line Revision

> Carry `(min, max)` range and validate every node.

# Problem 3 — LC226 Invert Binary Tree

## Recognition Signals

* Mirror Tree
* Invert Tree
* Flip Tree

Think

> Return the inverted subtree root.

---

## Core Intuition

Each recursive call inverts its own subtree.

Parent reconnects the inverted children by swapping them.

---

## Recursive Formula

```text
Invert Left

↓

Invert Right

↓

Swap

↓

Return Root
```

---

## Optimal Java

```java
class Solution {

    public TreeNode invertTree(TreeNode root) {

        if (root == null)
            return null;

        TreeNode left = invertTree(root.left);
        TreeNode right = invertTree(root.right);

        root.left = right;
        root.right = left;

        return root;
    }
}
```

---

## Complexity

```text
Time  : O(n)

Space : O(h)
```

---

## Interview Tip

Return the root of the transformed subtree so the parent can reconnect it.

---

## One-Line Revision

> Invert children → Swap → Return root.

---

# Problem 4 — LC617 Merge Two Binary Trees

## Recognition Signals

* Merge Trees
* Combine Trees
* Construct Tree

Think

> Return the merged subtree root.

---

## Core Intuition

There are only three cases:

```text
Both null

↓

Return null
```

```text
One null

↓

Return the existing subtree
```

```text
Both exist

↓

Merge values

↓

Merge children

↓

Return root
```

---

## Optimal Java

```java
class Solution {

    public TreeNode mergeTrees(TreeNode root1,
                               TreeNode root2) {

        if (root1 == null)
            return root2;

        if (root2 == null)
            return root1;

        root1.val += root2.val;

        root1.left =
                mergeTrees(root1.left,
                           root2.left);

        root1.right =
                mergeTrees(root1.right,
                           root2.right);

        return root1;
    }
}
```

---

## Complexity

```text
Time  : O(n)

Space : O(h)
```

---

## Interview Tip

Every recursive call returns the root of the merged subtree.

---

## One-Line Revision

> Merge children → Attach → Return merged root.

# Pattern Summary

## Validation Problems

Children return information that helps the parent validate the subtree.

Examples

* LC110 Balanced Binary Tree
* LC98 Validate BST

---

## Transformation Problems

Children return transformed subtree roots.

Parent reconnects them.

Examples

* LC226 Invert Binary Tree
* LC617 Merge Two Binary Trees

# Variation 1 vs Variation 2 vs Variation 3

| Variation                       | Main Idea                   | Information Flow                                            |
| ------------------------------- | --------------------------- | ----------------------------------------------------------- |
| Return Value DFS                | Compute subtree information | Bottom-Up                                                   |
| Root-to-Leaf DFS                | Carry state                 | Top-Down                                                    |
| Validation & Transformation DFS | Validate or modify subtree  | Mostly Bottom-Up (BST validation uses Top-Down constraints) |

This variation teaches how recursive calls return **useful subtree information** or **transformed subtree roots**.

You learned four high-frequency interview patterns:

* **Sentinel return value** (`-1`) for early termination.
* **Range propagation** for BST validation.
* **Recursive tree transformation**.
* **Recursive tree construction**.

These techniques form the foundation for many Medium and Hard tree interview problems.
