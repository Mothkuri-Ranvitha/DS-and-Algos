# 03 - Merge K Sorted Structures

---

# Variation Definition

This variation involves merging multiple **already sorted** data structures efficiently.

The structures may be:

```text
K Sorted Linked Lists

K Sorted Arrays

K Sorted Streams

K Sorted Rows
```

Instead of repeatedly scanning all structures, we use a **Min Heap** to always pick the smallest available element.

---

# Core Intuition

Suppose we have:

```text
List 1 : 1 → 4 → 5

List 2 : 1 → 3 → 4

List 3 : 2 → 6
```

Goal:

```text
Merge into one sorted list.
```

Answer:

```text
1 → 1 → 2 → 3 → 4 → 4 → 5 → 6
```

---

## Brute Force

Collect every element.

Sort again.

Complexity:

```text
O(N log N)
```

where N is the total number of elements.

---

## Better Observation

Each list is **already sorted**.

Only the **first element** of every list can be the next smallest element.

Instead of storing all elements, we only store:

```text
Current Head of Every List
```

inside a Min Heap.

---

# Recognition Signals

Think Merge K Pattern when you see:

* Merge K Sorted Lists
* Merge K Arrays
* Merge Sorted Streams
* Merge Multiple Sorted Files

Keywords:

```text
Merge

K Sorted

Sorted Lists

Sorted Arrays

Streams
```

---

# Core Idea

Maintain a **Min Heap** containing one element from each sorted structure.

Whenever we remove the smallest element:

* Add it to the answer.
* Insert the next element from the same list.

Repeat until the heap becomes empty.

---

# Why Min Heap?

We always need:

```text
Smallest Available Element
```

So Min Heap gives:

```text
peek() → O(1)

poll() → O(log K)
```

where K is the number of lists.

# Problem 1

# LC23 - Merge K Sorted Lists

## Problem Description

Given:

```text
List1

1 → 4 → 5

List2

1 → 3 → 4

List3

2 → 6
```

Return:

```text
1 → 1 → 2 → 3 → 4 → 4 → 5 → 6
```

---

# Intuition

Initially:

Insert the first node of every list.

Heap:

```text
1
1
2
```

The heap always contains the smallest available node from every list.

---

Remove:

```text
1
```

Append it to answer.

Now insert:

```text
4
```

(the next node from that same list)

Heap becomes:

```text
1
2
4
```

Again remove the smallest.

Repeat until heap becomes empty.

# Algorithm

1. Create Min Heap based on node value.
2. Insert first node of every list.
3. While heap is not empty:

   * Remove smallest node.
   * Append it to answer.
   * If removed node has next node:

     * Insert next node into heap.
4. Return merged list.

---

# Code

```java
class Solution {

    public ListNode mergeKLists(ListNode[] lists) {

        PriorityQueue<ListNode> pq =
            new PriorityQueue<>(
                (a, b) -> a.val - b.val);

        for (ListNode node : lists) {
            if (node != null)
                pq.offer(node);
        }

        ListNode dummy = new ListNode(0);
        ListNode curr = dummy;

        while (!pq.isEmpty()) {

            ListNode node = pq.poll();

            curr.next = node;
            curr = curr.next;

            if (node.next != null)
                pq.offer(node.next);
        }

        return dummy.next;
    }
}
```

```text
Space : O(K)
```


# Problem 2

# LC373 - Find K Pairs with Smallest Sums

## Problem Description

Given:

```java
nums1 = [1,7,11]

nums2 = [2,4,6]

k = 3
```

Possible pairs:

```text
(1,2)=3

(1,4)=5

(1,6)=7

(7,2)=9

(7,4)=11

...
```

Return:

```text
[
(1,2),
(1,4),
(1,6)
]
```

---

# Intuition

Do NOT generate every pair.

That would be:

```text
O(M × N)
```

Instead:

Think like merging K sorted lists.

Each element in nums1 starts a sorted sequence:

```text
1:

(1,2)

(1,4)

(1,6)

----------------

7:

(7,2)

(7,4)

(7,6)

----------------

11:

(11,2)

(11,4)

(11,6)
```

Initially push:

```text
(1,2)

(7,2)

(11,2)
```

After removing:

```text
(1,2)
```

Insert:

```text
(1,4)
```

Continue until K pairs are produced.

---

# Algorithm

1. Insert first pair of every row.
2. Pop smallest sum.
3. Add answer.
4. Push next pair from same row.
5. Stop after K pairs.

---

# Code

```java
class Solution {

    public List<List<Integer>> kSmallestPairs(
            int[] nums1,
            int[] nums2,
            int k) {

        List<List<Integer>> ans =
                new ArrayList<>();

        if (nums1.length == 0 ||
            nums2.length == 0)
            return ans;

        PriorityQueue<int[]> pq =
            new PriorityQueue<>(
                (a, b) ->
                (nums1[a[0]] + nums2[a[1]]) -
                (nums1[b[0]] + nums2[b[1]]));

        for (int i = 0;
             i < Math.min(nums1.length, k);
             i++) {

            pq.offer(new int[]{i, 0});
        }

        while (k-- > 0 && !pq.isEmpty()) {

            int[] cur = pq.poll();

            int i = cur[0];
            int j = cur[1];

            ans.add(Arrays.asList(
                    nums1[i],
                    nums2[j]));

            if (j + 1 < nums2.length) {

                pq.offer(new int[]{i, j + 1});
            }
        }

        return ans;
    }
}
```

---

```text

Space : O(K)
```
