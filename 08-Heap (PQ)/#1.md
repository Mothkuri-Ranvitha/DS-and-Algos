# Variation Definition

Top K Problems ask us to find:

```text
K Largest Elements

K Smallest Elements

K Most Frequent Elements

K Highest Priority Elements
```

without sorting the entire array.

---

Suppose:

```java
nums = [3,2,1,5,6,4]

k = 2
```

Question:

```text
Find 2nd Largest Element

```



Observation:


We only need:


# Problem 1

# LC215 - Kth Largest Element in an Array

## Problem Description

Given:

```java
nums = [3,2,1,5,6,4]

k = 2
```

Sorted:

```java
1 2 3 4 5 6
```

2nd Largest:

```java
5
```

Return:

```java
5
```

---

## Intuition

We only care about:

```text
Largest K Elements
```

Not entire sorting.

---

Maintain:

```text
Min Heap of Size K
```

---

Heap always stores:

```text
Current Top K Largest Elements
```

---

Heap Top:

```text
Kth Largest
```

---

---

## Algorithm

1. Create Min Heap.
2. Insert every element.
3. If heap size exceeds K:

   * remove smallest.
4. Heap top becomes answer.

---

## Code

```java
class Solution {

    public int findKthLargest(
            int[] nums,
            int k) {

        PriorityQueue<Integer> pq =
                new PriorityQueue<>();

        for(int num : nums){

            pq.offer(num);

            if(pq.size() > k){

                pq.poll();
            }
        }

        return pq.peek();
    }
}
```

---

## Complexity

```text
Time  : O(N log K)

Space : O(K)
```

# Problem 2

# LC347 - Top K Frequent Elements

## Problem Description

Given:

```java
nums = [1,1,1,2,2,3]

k = 2
```

Frequencies:

```text
1 → 3

2 → 2

3 → 1
```

Top 2 Frequent:

```java
[1,2]
```

---

## Intuition

Need:

```text
Highest Frequencies
```

First count frequency.

---

Create:

```java
HashMap
```

```text
1 -> 3

2 -> 2

3 -> 1
```

---

Now keep:

```text
Top K Frequencies
```

inside Min Heap.

---

Heap stores:

```java
[number, frequency]
```

---

Heap Top:

```text
Smallest Frequency
```

among Top-K.

---

# Dry Run

Map:

```text
1 -> 3

2 -> 2

3 -> 1
```

---

Insert:

```text
(1,3)
```

Heap:

```text
[(1,3)]
```

---

Insert:

```text
(2,2)
```

Heap:

```text
[(2,2),(1,3)]
```

---

Insert:

```text
(3,1)
```

Heap:

```text
[(3,1),(1,3),(2,2)]
```

Size > K

Remove:

```text
(3,1)
```

Heap:

```text
[(2,2),(1,3)]
```

---

Answer:

```java
[1,2]
```

---

## Algorithm

1. Count frequencies.
2. Create Min Heap by frequency.
3. Insert each map entry.
4. If heap size > K:

   * remove smallest frequency.
5. Extract heap elements.

---

## Code

```java
class Solution {

    public int[] topKFrequent(
            int[] nums,
            int k) {

        Map<Integer,Integer> freq =
                new HashMap<>();

        for(int num : nums){

            freq.put(
                num,
                freq.getOrDefault(
                    num,0)+1);
        }

        PriorityQueue<int[]> pq =
            new PriorityQueue<>(
                (a,b)->a[1]-b[1]);

        for(int key : freq.keySet()){

            pq.offer(
                new int[]{
                    key,
                    freq.get(key)
                });

            if(pq.size() > k){

                pq.poll();
            }
        }

        int[] ans = new int[k];

        for(int i=k-1;i>=0;i--){

            ans[i] = pq.poll()[0];
        }

        return ans;
    }
}
```

---

## Complexity

```text
Time  : O(N log K)

Space : O(N)
```
