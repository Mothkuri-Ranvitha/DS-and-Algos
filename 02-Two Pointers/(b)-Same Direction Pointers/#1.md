# Fast-Slow Compression

> Keep useful values. Ignore unwanted values.

---

## Core Idea

```text
Fast Reads

↓

Useful?

↓

Slow Writes
```

Fast scans the array.

Slow maintains the valid portion.

---

## Recognition Signals

* Remove duplicates
* Remove element
* Return new length
* In-place modification
* O(1) extra space

---

## Generic Template

```java
int slow = 0;

for(int fast = 0; fast < nums.length; fast++) {

    if(valid(nums[fast])) {
        nums[slow] = nums[fast];
        slow++;
    }
}
```

## Example Problems

### LC26 - Remove Duplicates from Sorted Array

Example:

```text
[1,1,2,2,3]

↓

[1,2,3]
```

```java
class Solution {
    public int removeDuplicates(int[] nums) {

        int slow = 1;

        for(int fast = 1; fast < nums.length; fast++) {

            if(nums[fast] != nums[fast - 1]) {
                nums[slow] = nums[fast];
                slow++;
            }
        }

        return slow;
    }
}
```

Time: O(N)

Space: O(1)

---

### LC27 - Remove Element

Example:

```text
[3,2,2,3]

remove = 3

↓

[2,2]
```

```java
class Solution {
    public int removeElement(int[] nums, int val) {

        int slow = 0;

        for(int fast = 0; fast < nums.length; fast++) {

            if(nums[fast] != val) {
                nums[slow] = nums[fast];
                slow++;
            }
        }

        return slow;
    }
}
```

Time: O(N)

Space: O(1)
