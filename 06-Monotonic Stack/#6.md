#  Contribution Problems

## Variation Definition

In previous Monotonic Stack problems, we asked:

* Next Greater Element
* Previous Smaller Element
* Distance
* Span
* Rectangle Area

Contribution Problems ask a completely different question:

```text
How many subarrays use this element
as their minimum or maximum?
```

Instead of finding:

```text
Next Greater

Next Smaller

Area
```

we calculate:

```text
Contribution of each element
to the final answer.
```

This is one of the most important advanced Monotonic Stack patterns.

---

# Why Does This Variation Exist?

Consider:

```text
nums = [3,1,2,4]
```

All subarrays:

```text
[3]
[3,1]
[3,1,2]
[3,1,2,4]

[1]
[1,2]
[1,2,4]

[2]
[2,4]

[4]
```

Minimums:

```text
3
1
1
1

1
1
1

2
2

4
```

Sum:

```text
17
```

Brute force:

```text
Generate all subarrays
Find minimum for each
```

Complexity:

```text
O(N²) or O(N³)
```

Too slow.

# Core Intuition

Instead of asking:

```text
What is the minimum
of every subarray?
```

Ask:

```text
For each element,

how many subarrays
consider me as minimum?
```

This completely changes the problem.

---

```text
[3,1,2,4]
```


```text

[3,1,2]

[3,1,2,4]
```


Contribution:


# Key Observation

For every element:

```text
Find how many choices exist
on the left.

Find how many choices exist
on the right.
```

Then:

```text
Contribution

=
value
×
leftChoices
×
rightChoices
```

# Why Previous Smaller & Next Smaller?

Suppose:

```text
nums = [3,1,2,4]
```

For:

```text
1
```

Previous Smaller:

```text
None
```

Next Smaller:

```text
None
```

Therefore:

```text
1 can expand
through the entire array.
```

Because:

```text
No smaller element blocks it.
```

---

# Formula

For minimum contribution:

```java
contribution =
nums[i]
*
leftCount
*
rightCount;
```

Where:

```java
leftCount =
i - previousSmallerIndex;
```

```java
rightCount =
nextSmallerIndex - i;
```

# Algorithm

For every element:

### Step 1

Find:

```text
Previous Smaller Element
```

---

### Step 2

Find:

```text
Next Smaller Element
```

---

### Step 3

Calculate:

```java
left =
i - prevSmaller
```

```java
right =
nextSmaller - i
```

---

### Step 4

Add contribution:

```java
nums[i] * left * right
```

---

# Code

```java
class Solution {

    public int sumSubarrayMins(int[] arr) {

        int n = arr.length;

        long ans = 0;

        int MOD = 1_000_000_007;

        int[] left = new int[n];
        int[] right = new int[n];

        Stack<Integer> st =
                new Stack<>();

        for(int i=0;i<n;i++){

            while(!st.isEmpty()
                    && arr[st.peek()] > arr[i]){

                st.pop();
            }

            left[i] =
                    st.isEmpty()
                    ? -1
                    : st.peek();

            st.push(i);
        }

        st.clear();

        for(int i=n-1;i>=0;i--){

            while(!st.isEmpty()
                    && arr[st.peek()] >= arr[i]){

                st.pop();
            }

            right[i] =
                    st.isEmpty()
                    ? n
                    : st.peek();

            st.push(i);
        }

        for(int i=0;i<n;i++){

            long leftCount =
                    i - left[i];

            long rightCount =
                    right[i] - i;

            ans +=
                    (long)arr[i]
                    * leftCount
                    * rightCount;

            ans %= MOD;
        }

        return (int)ans;
    }
}
```

---

# Why Use Different Comparisons?

Left Side:

```java
>
```

Right Side:

```java
>=
```

This avoids:

```text
Duplicate counting
```

when equal elements exist.

This is one of the most important interview observations.

# Difference From Rectangle Problems

| Rectangle Problems | Contribution Problems        |
| ------------------ | ---------------------------- |
| Width × Height     | Left Choices × Right Choices |
| Area               | Contribution                 |
| Single Rectangle   | All Subarrays                |
