# Removal / Deletion Problems

## Variation Definition

In previous Monotonic Stack problems, we used the stack to find:

* Next Greater Element
* Previous Smaller Element
* Boundaries
* Contributions
* Areas

In Removal Problems, we use the stack to:

```text
Remove bad choices
to build the best possible answer.
```

Instead of asking:

```text
What is the next greater element?
```

we ask:

```text
Can I remove the current element
to get a better final result?
```

---

# Why Does This Variation Exist?

Consider:

```text
num = "1432219"

k = 3
```

We need:

```text
Remove exactly 3 digits
```

to create:

```text
Smallest Possible Number
```

Answer:

```text
1219
```

Question:

```text
How do we know
which digits to remove?
```

---

# Core Intuition

Suppose:

```text
1 4 3
```

When:

```text
3
```

arrives,

we already have:

```text
4
```

inside stack.

Question:

```text
Would the answer become smaller
if we remove 4 and keep 3?
```

Yes.

Because:

```text
3 < 4
```

Therefore:

```text
Remove 4
```

This is the heart of this pattern.

---

# Generic Rule

Whenever:

```text
Current Element
is better than
Stack Top
```

and

```text
Removal Budget Available
```

we remove the stack top.

# Main Idea

```java
while(
    stack not empty
    &&
    top is worse
    &&
    removal available
)
{
    pop();
}
```

Push current element.

This creates the optimal answer greedily.

# Main Problem

## LC402 - Remove K Digits

### Problem

Remove exactly:

```text
k
```

digits to obtain the smallest possible number.

---

### Example

Input:

```text
num = "1432219"
k = 3
```

Output:

```text
1219
```

---

## Why Monotonic Increasing Stack?

We want:

```text
Smallest Number
```

Therefore:

```text
Smaller digits should appear earlier.
```

Maintain:

```text
Increasing Stack
```

Whenever:

```text
current digit < stack top
```

remove stack top.

# Generic Template

```java
while(!st.isEmpty()
        &&
        st.peek() > current
        &&
        k > 0){

    st.pop();
    k--;
}
```

Push current.

# Difference From Contribution Problems

| Contribution Problems | Removal Problems      |
| --------------------- | --------------------- |
| Count Subarrays       | Build Best Answer     |
| Previous/Next Smaller | Remove Worse Elements |
| Mathematical Counting | Greedy Construction   |
