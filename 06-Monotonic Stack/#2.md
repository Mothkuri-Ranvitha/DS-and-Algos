# 02 - Circular Array Problems

## Variation Definition

In normal arrays, when we reach the last element, the search stops.

In circular arrays, after reaching the last element, we continue searching from the beginning.

Example:

```text
Array:

[1,2,1]

Circular View:

1 → 2 → 1 → 1 → 2 → ...
```

This variation is commonly used when the problem asks:

```text
Find next greater element in a circular array.
```

---

# Why Does This Variation Exist?

Consider:

```text
[1,2,1]
```

For the last element:

```text
1
```

The next greater element is:

```text
2
```

But:

```text
2
```

appears before it in the array.

A normal Next Greater Element solution would stop at the end and miss this answer.

Therefore we need a circular traversal.

# Core Intuition

Normal NGE:

```text
Search Right
```

Circular NGE:

```text
Search Right

If not found

Continue from beginning
```

Instead of physically creating:

```text
[1,2,1,1,2,1]
```

we simulate it using:

```java
i % n
```

# Main Problem

## LC503 - Next Greater Element II

### Problem

Given a circular array, return the next greater element for every position.

If no greater element exists:

```text
return -1
```

---

### Example 1

Input:

```text
[1,2,1]
```

Output:

```text
[2,-1,2]
```

Explanation:

```text
1 → 2

2 → No Greater

1 → 2
```

---

### Example 2

Input:

```text
[3,8,4,1,2]
```

Output:

```text
[8,-1,8,2,3]
```

---

# Brute Force

For every element:

```java
Search N positions
```

Complexity:

```text
O(N²)
```

---

# Optimal Intuition

Use the same idea as Next Greater Element.

Maintain:

```text
Monotonic Decreasing Stack
```

The only change:

```text
Traverse Array Twice
```

Why?

Because every element should get a chance to look beyond the end of the array.

---

# Key Trick

Instead of:

```java
for(int i=n-1;i>=0;i--)
```

Use:

```java
for(int i=2*n-1;i>=0;i--)
```

Access actual element using:

```java
nums[i%n]
```

This simulates:

```text
Array + Array
```

without extra space.

# Code

```java
class Solution {

    public int[] nextGreaterElements(int[] nums) {

        int n = nums.length;

        int[] ans = new int[n];

        Stack<Integer> st = new Stack<>();

        for(int i = 2*n - 1; i >= 0; i--) {

            while(!st.isEmpty()
                    && st.peek() <= nums[i % n]) {

                st.pop();
            }

            if(i < n) {

                ans[i] =
                        st.isEmpty()
                        ? -1
                        : st.peek();
            }

            st.push(nums[i % n]);
        }

        return ans;
    }
}
```

# Why Does This Work?

For every element:

```text
First Pass
→ Build future information

Second Pass
→ Answer queries
```

Every element effectively sees:

```text
Its entire circular right side
```
