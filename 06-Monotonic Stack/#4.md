# 04 - Span Problems

## Variation Definition

In Nearest Greater/Smaller problems, we find:

```text
Next Greater

Next Smaller

Previous Greater

Previous Smaller
```

But in Span Problems, we ask:

```text
How many consecutive elements
satisfy a condition?
```

Instead of returning:

```text
Value
```

or

```text
Index
```

we return:

```text
Count
```

---

# Why Does This Variation Exist?

Consider stock prices:

```text
[100,80,60,70,60,75,85]
```

For:

```text
85
```

Question:

```text
How many consecutive previous days
had price <= 85 ?
```

Answer:

```text
6
```

because:

```text
80
60
70
60
75
```

all satisfy the condition.

So we need:

```text
Continuous Count
```

not a next/previous element.

# Understanding Span

For:

```text
85
```

Previous prices:

```text
100 80 60 70 60 75
```

Looking backwards:

```text
75 <= 85
60 <= 85
70 <= 85
60 <= 85
80 <= 85
```

All valid.

Stop at:

```text
100
```

because:

```text
100 > 85
```

Span:

```text
6
```

---

# Brute Force

For every price:

```java
Move left

Count valid elements
```

Complexity:

```text
O(N²)
```

---

# Optimal Intuition

Observe:

```text
Smaller prices become useless
when a larger price arrives.
```

Example:

```text
80 60 70
```

When:

```text
70
```

arrives:

```text
60
```

can never block future prices.

So remove it.

This is exactly:

```text
Monotonic Stack
```

---

# Key Idea

Instead of storing:

```java
price
```

Store:

```java
(price, span)
```

Because when:

```text
75
```

removes:

```text
60
70
```

their spans can be merged directly.

# Algorithm

For every new price:

### Step 1

Initialize:

```java
span = 1;
```

---

### Step 2

Remove all smaller/equal prices.

```java
while(topPrice <= currentPrice)
```

---

### Step 3

Add removed spans.

```java
span += poppedSpan;
```

---

### Step 4

Push:

```java
(currentPrice, span)
```

---

### Step 5

Return span.

---

# Code

```java
class StockSpanner {

    Stack<int[]> st;

    public StockSpanner() {

        st = new Stack<>();
    }

    public int next(int price) {

        int span = 1;

        while(!st.isEmpty()
                && st.peek()[0] <= price) {

            span += st.pop()[1];
        }

        st.push(
                new int[]{
                        price,
                        span
                });

        return span;
    }
}
```

---

# Why Store Span?

Without storing span:

```text
Need to recount
all previous elements
```

every time.

That becomes:

```text
O(N²)
```

With:

```text
(price, span)
```

we reuse previous work.

---

# Complexity

### Time

```text

```

Each element:

```text

Popped Once
```

### Space

