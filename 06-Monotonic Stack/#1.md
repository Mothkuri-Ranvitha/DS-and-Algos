# 01 - Nearest Greater / Smaller Elements

## Variation Definition

This is the foundation of the entire Monotonic Stack pattern.

In these problems, for every element, we need to find:

* Next Greater Element
* Next Smaller Element
* Previous Greater Element
* Previous Smaller Element

Most advanced Monotonic Stack problems are built on these four concepts.

---

# Why Does This Variation Exist?

Suppose:

```text
[4, 5, 2, 10]
```

For element:

```text
2
```

Questions could be:

```text
Next Greater ?
→ 10

Next Smaller ?
→ None

Previous Greater ?
→ 5

Previous Smaller ?
→ None
```

Brute Force:

```java
for(int i=0;i<n;i++)
    search left/right
```

Complexity:

```text
O(N²)
```

Monotonic Stack:

```text
O(N)
```

# Type 1 — Next Greater Element

## Problem

For every element, find the first greater element on the right.

Example:

```text
Input

[2,1,3]

Output

[3,3,-1]
```

Explanation:

```text
2 → 3

1 → 3

3 → No Greater Element
```

---

## Intuition

Current element wants:

```text
Nearest Greater Element
on Right Side
```

Process array from:

```text
Right → Left
```

Maintain:

```text
Decreasing Stack
```

Why?

Because smaller elements can never become the answer.

---

## Dry Run

Input:

```text
[2,1,3]
```

Start from right.

### Element = 3

Stack:

```text
[]
```

Answer:

```text
-1
```

Push:

```text
[3]
```

---

### Element = 1

Stack:

```text
[3]
```

Top:

```text
3
```

Answer:

```text
3
```

Push:

```text
[3,1]
```

---

### Element = 2

Remove:

```text
1
```

because:

```text
1 < 2
```

Stack:

```text
[3]
```

Answer:

```text
3
```

Final:

```text
[3,3,-1]
```

---

## Code

```java
public int[] nextGreater(int[] nums){

    int n = nums.length;

    int[] ans = new int[n];

    Stack<Integer> st =
            new Stack<>();

    for(int i=n-1;i>=0;i--){

        while(!st.isEmpty()
                && st.peek()<=nums[i]){

            st.pop();
        }

        ans[i] =
                st.isEmpty()
                ? -1
                : st.peek();

        st.push(nums[i]);
    }

    return ans;
}
```

---

## Complexity

```text
Time  : O(N)

Space : O(N)
```

# Important Observation

| Problem          | Direction    | Stack Type |
| ---------------- | ------------ | ---------- |
| Next Greater     | Right → Left | Decreasing |
| Next Smaller     | Right → Left | Increasing |
| Previous Greater | Left → Right | Decreasing |
| Previous Smaller | Left → Right | Increasing |
