# 05 - Boundary Problems

## Variation Definition

In previous Monotonic Stack variations, we found:

* Next Greater Element
* Previous Greater Element
* Distance to Next Greater
* Span Count

In Boundary Problems, we ask:

```text
How far can an element expand
towards left and right?
```

Instead of finding:

```text
Value
Index
Count
```

we find:

```text
Left Boundary
Right Boundary
```

for every element.

---

## Why Does This Variation Exist?

Consider histogram bars:

```text
[2,1,5,6,2,3]
```

For bar:

```text
5
```

Question:

```text
How far can height 5 expand?
```

Move left:

```text
1
```

Stop because:

```text
1 < 5
```

Move right:

```text
6
```

Valid.

Next:

```text
2
```

Stop because:

```text
2 < 5
```

Therefore:

```text
Left Boundary = index 2
Right Boundary = index 3
```

This boundary information is used to calculate area.

# Core Intuition

For every element:

```text
Find Previous Smaller
Find Next Smaller
```

Why smaller?

Because:

```text
Smaller element blocks expansion.
```

The first smaller element on both sides determines the maximum range.

---

## Main Problem

### Previous Smaller Element

Find nearest smaller element on left.

Example:

```text
[2,1,5,6,2,3]
```

Output Indices:

```text
[-1,-1,1,2,1,4]
```

---

### Next Smaller Element

Find nearest smaller element on right.

Example:

```text
[2,1,5,6,2,3]
```

Output Indices:

```text
[1,6,4,4,6,6]
```

Here:

```text
n = 6
```

is used when no smaller element exists.

---

## Why Store Indices?

We need:

```text
Width
```

and width requires:

```java
rightBoundary - leftBoundary - 1
```

Therefore:

```text
Store Indices
```

not values.

---

# Building Left Boundary

## Previous Smaller Element

### Algorithm

Process:

```text
Left → Right
```

Maintain:

```text
Increasing Stack
```

Remove:

```text
>= current element
```

because they cannot become smaller boundaries.

---

### Code

```java
public int[] previousSmaller(int[] heights){

    int n = heights.length;

    int[] left = new int[n];

    Stack<Integer> st =
            new Stack<>();

    for(int i=0;i<n;i++){

        while(!st.isEmpty()
                &&
                heights[st.peek()]
                >= heights[i]){

            st.pop();
        }

        left[i] =
                st.isEmpty()
                ? -1
                : st.peek();

        st.push(i);
    }

    return left;
}
```

---

### Complexity

```text
Time  : O(N)

Space : O(N)
```

---

# Building Right Boundary

## Next Smaller Element

### Algorithm

Process:

```text
Right → Left
```

Maintain:

```text
Increasing Stack
```

Remove:

```text
>= current element
```

---

### Code

```java
public int[] nextSmaller(int[] heights){

    int n = heights.length;

    int[] right = new int[n];

    Stack<Integer> st =
            new Stack<>();

    for(int i=n-1;i>=0;i--){

        while(!st.isEmpty()
                &&
                heights[st.peek()]
                >= heights[i]){

            st.pop();
        }

        right[i] =
                st.isEmpty()
                ? n
                : st.peek();

        st.push(i);
    }

    return right;
}
```

---

### Complexity

```text
Time  : O(N)

Space : O(N)
```

# Width Formula

Once boundaries are known:

```java
width =
right[i] - left[i] - 1;
```

Example:

```text
Height = 5

Left = 1

Right = 4
```

Width:

```java
4 - 1 - 1
=
2
```

Valid bars:

```text
5 6
```
