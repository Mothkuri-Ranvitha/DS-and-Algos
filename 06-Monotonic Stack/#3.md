# 03 - Distance Based Problems

## Variation Definition

In Nearest Greater/Smaller problems, we return the value of the answer.

Example:

```text
[2,1,3]

Output

[3,3,-1]
```

Here we return:

```text
Next Greater Value
```

---

In Distance Based Problems, we return:

```text
Distance to Answer
```

instead of the actual value.

Example:

```text
Temperature Array

[73,74,75,71,69,72,76,73]
```

Output:

```text
[1,1,4,2,1,1,0,0]
```

Meaning:

```text
73 → warmer day after 1 day

74 → warmer day after 1 day

75 → warmer day after 4 days
```

We need distance.

Not the temperature itself.

---

# Why Does This Variation Exist?

Many interview problems ask:

```text
How far?

How many days?

How many positions?

Distance to next greater?
```

Instead of:

```text
What is next greater?
```

Because of this:

```text
Store Index

instead of

Store Value
```

---

# Recognition Signals

Think of Distance Based Monotonic Stack when you see:


Keywords:


Normal Next Greater:

```text

```

Distance Problems:

```text

```

```java

```

# Main Problem

## LC739 - Daily Temperatures

### Problem

Given daily temperatures:

Return how many days you must wait until a warmer temperature.

If no warmer temperature exists:

```text
return 0
```

---

### Example

Input:

```text
[73,74,75,71,69,72,76,73]
```

Output:

```text
[1,1,4,2,1,1,0,0]
```

---

### Explanation

```text
73 → 74 after 1 day

74 → 75 after 1 day

75 → 76 after 4 days

71 → 72 after 2 days

69 → 72 after 1 day

72 → 76 after 1 day

76 → none

73 → none
```

---

# Algorithm

For each temperature:

### Step 1

Remove all temperatures:

```text
<= current temperature
```

because they can never be the answer.

---

### Step 2

Top of stack becomes:

```text
Next Greater Temperature
```

---

### Step 3

Compute distance:

```java
topIndex - currentIndex
```

---

### Step 4

Push current index.

---

# Code

```java
class Solution {

    public int[] dailyTemperatures(int[] temperatures) {

        int n = temperatures.length;

        int[] ans = new int[n];

        Stack<Integer> st =
                new Stack<>();

        for(int i=n-1;i>=0;i--) {

            while(!st.isEmpty()
                    &&
                    temperatures[st.peek()]
                    <= temperatures[i]) {

                st.pop();
            }

            ans[i] =
                    st.isEmpty()
                    ? 0
                    : st.peek() - i;

            st.push(i);
        }

        return ans;
    }
}
```

# Why Store Indices?

Wrong:

```java
Stack<Integer> values
```

Because:

```text
Need distance
```

Distance requires:

```text
Position
```

Therefore:

```java
Stack<Integer> indices
```
