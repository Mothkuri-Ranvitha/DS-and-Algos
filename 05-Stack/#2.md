# Expression Evaluation

> Use a Stack to evaluate mathematical expressions.

---

# Definition

Expression Evaluation problems require computing the result of an arithmetic expression while respecting operator order.

Examples:

```text
2 + 3

2 + 3 * 4

(2 + 3) * 4

2 1 + 3 *
```

---

# Why Stack?

When evaluating an expression, some operations must wait.

Example:

```text
2 + 3 * 4
```

We cannot immediately calculate:

```text
2 + 3
```

because:

```text
*
```

has higher precedence.

We need temporary storage.

Stack provides that storage.

---

# Core Intuition

Think:

```text
Some numbers/operators
must wait until enough
information becomes available.
```

Store them in a stack.

Process later.

# LC150 - Evaluate Reverse Polish Notation

## Problem

Evaluate the value of an arithmetic expression in Reverse Polish Notation.

Valid operators:

```text
+
-
*
/
```

Example:

```text
["2","1","+","3","*"]

((2 + 1) * 3)

Answer = 9
```

---

## Algorithm

1. Create Stack<Integer>.
2. Traverse tokens.
3. If number:

   * Push into stack.
4. If operator:

   * Pop two numbers.
   * Perform operation.
   * Push result back.
5. Final stack top is answer.

---

## Logic

Operator always uses the two most recent numbers.

Example:

```text
2 1 +
```

When '+' appears:

```text
Pop 1
Pop 2

2 + 1 = 3

Push 3
```

---

## Code

```java
class Solution {

    public int evalRPN(String[] tokens) {

        Stack<Integer> st = new Stack<>();

        for(String token : tokens){

            if(token.equals("+")){

                int b = st.pop();
                int a = st.pop();

                st.push(a + b);
            }

            else if(token.equals("-")){

                int b = st.pop();
                int a = st.pop();

                st.push(a - b);
            }

            else if(token.equals("*")){

                int b = st.pop();
                int a = st.pop();

                st.push(a * b);
            }

            else if(token.equals("/")){

                int b = st.pop();
                int a = st.pop();

                st.push(a / b);
            }

            else{

                st.push(Integer.parseInt(token));
            }
        }

        return st.peek();
    }
}
```

---

## Complexity

Time: O(N)

Space: O(N)
# LC224 - Basic Calculator

## Problem

Evaluate a mathematical expression containing:

```text
+
-

(
)
```

Example:

```text
1 + (2 - 3)

Answer = 0
```

---

## Algorithm

1. Maintain:

   * result
   * sign
2. Traverse expression.
3. When '(':

   * Save current result and sign in stack.
4. When ')':

   * Complete sub-expression.
5. Continue evaluation.

---

## Logic

Parentheses create nested calculations.

Example:

```text
1 + (2 - 3)
```

Before entering:

```text
( )
```

store current state.

After exiting:

```text
Resume previous state.
```

Stack naturally handles nested expressions.

---

## Code

```java
class Solution {

    public int calculate(String s) {

        Stack<Integer> st = new Stack<>();

        int result = 0;
        int number = 0;
        int sign = 1;

        for(int i=0;i<s.length();i++){

            char ch = s.charAt(i);

            if(Character.isDigit(ch)){

                number =
                    number * 10 +
                    (ch - '0');
            }

            else if(ch == '+'){

                result += sign * number;
                number = 0;
                sign = 1;
            }

            else if(ch == '-'){

                result += sign * number;
                number = 0;
                sign = -1;
            }

            else if(ch == '('){

                st.push(result);
                st.push(sign);

                result = 0;
                sign = 1;
            }

            else if(ch == ')'){

                result += sign * number;
                number = 0;

                result *= st.pop();
                result += st.pop();
            }
        }

        return result + sign * number;
    }
}
```

---

## Complexity

Time: O(N)

Space: O(N)
# LC227 - Basic Calculator II

## Problem

Given a string expression containing:

```text
+
-
*
/
```

and non-negative integers, evaluate the expression and return the result.

Rules:

* Integer division truncates toward zero.
* No parentheses are present.
* Expression is always valid.

### Example 1

```text
Input: "3+2*2"

Output: 7
```

### Example 2

```text
Input: " 3/2 "

Output: 1
```

### Example 3

```text
Input: " 3+5 / 2 "

Output: 5
```

---

# Core Idea

The challenge is:

```text
* and /
have higher precedence than
+ and -
```

Example:

```text
2 + 3 * 4
```

Wrong:

```text
(2 + 3) * 4 = 20
```

Correct:

```text
2 + (3 * 4) = 14
```

So we cannot simply evaluate from left to right.

---

# Algorithm

Use a stack.

Maintain:

```java
number
operator
stack
```

Traverse expression.

### If digit

Build current number.

### If operator or end of string

Process previous operator.

---

## Previous Operator = '+'

Push number.

```java
stack.push(number);
```

---

## Previous Operator = '-'

Push negative number.

```java
stack.push(-number);
```

---

## Previous Operator = '*'

Calculate immediately.

```java
stack.push(stack.pop() * number);
```

---

## Previous Operator = '/'

Calculate immediately.

```java
stack.push(stack.pop() / number);
```

---

After traversal:

Add all values in stack.

That gives final answer.

---

# Why Does This Work?

For:

```text
3 + 2 * 2
```

When we reach:

```text
*
```

we already know:

```text
2 * 2
```

must happen before addition.

So instead of pushing:

```text
2
```

we replace it with:

```text
4
```

Stack becomes:

```text
[3,4]
```

Now simple sum gives:

```text
7
```

# Java Code

```java
class Solution {

    public int calculate(String s) {

        Stack<Integer> stack = new Stack<>();

        int number = 0;
        char operator = '+';

        for(int i = 0; i < s.length(); i++) {

            char ch = s.charAt(i);

            if(Character.isDigit(ch)) {

                number = number * 10 + (ch - '0');
            }

            // Process the current number when:
            // 1. We encounter an operator (+, -, *, /), or
            // 2. We reach the last character of the string.
            //
            // Example: "3+2*2"
            // When we reach '+', we know the number 3 is complete,
            // so we process it using the previous operator.
            //
            // The second condition (i == s.length() - 1) is important
            // because the last number may not be followed by an operator.
            // Without it, the final number would never be processed.
            if((!Character.isDigit(ch) && ch != ' ')
                    || i == s.length() - 1) {

                if(operator == '+') {

                    stack.push(number);
                }

                else if(operator == '-') {

                    stack.push(-number);
                }

                else if(operator == '*') {

                    stack.push(stack.pop() * number);
                }

                else if(operator == '/') {

                    stack.push(stack.pop() / number);
                }

                operator = ch;
                number = 0;
            }
        }

        int result = 0;

        while(!stack.isEmpty()) {

            result += stack.pop();
        }

        return result;
    }
}
```

# Interview Explanation

"We use a stack to handle operator precedence. Addition and subtraction are deferred by pushing numbers into the stack, while multiplication and division are evaluated immediately with the top stack element. At the end, summing the stack gives the final result."
