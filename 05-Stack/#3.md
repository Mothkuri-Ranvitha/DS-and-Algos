# Stack Simulation

## Definition

Stack Simulation problems use a stack to imitate a real-world process.

The goal is not necessarily to find matching brackets or evaluate expressions.

Instead, we use a stack because the process naturally follows:

```text
Last In
First Out

(LIFO)
```

---

# Core Intuition

Think:

```text
Something happened recently.

A future operation needs to
use, remove, undo, compare,
or interact with that recent item.
```

The most recent item is always important.

Therefore:

```text
Use Stack
```

---

# Why Does This Pattern Exist?

Many systems work like:

```text
Undo Operations

Browser History

Backspace

Text Editors

Collisions

Push / Pop Operations

Function Calls
```

The latest action is processed first.

That is exactly Stack behavior.

# Variation 1

## LC1047 - Remove All Adjacent Duplicates In String

### Problem

Remove adjacent duplicate characters repeatedly until none remain.

Example:

```text
Input:

abbaca

Output:

ca
```

---

### Intuition

If current character equals stack top:

```text
Remove both
```

Otherwise:

```text
Push current character
```

Stack always contains the current valid string.

---

### Algorithm

```text
For each character

Same as stack top?
    Pop

Different?
    Push
```

---

### Code

```java
class Solution {

    public String removeDuplicates(String s) {

        Stack<Character> st = new Stack<>();

        for(char ch : s.toCharArray()) {

            if(!st.isEmpty() &&
               st.peek() == ch) {

                st.pop();
            }

            else {

                st.push(ch);
            }
        }

        StringBuilder ans =
                new StringBuilder();

        for(char ch : st) {

            ans.append(ch);
        }

        return ans.toString();
    }
}
```

### Complexity

```text
Time  : O(N)

Space : O(N)
```

# Variation 2

## LC844 - Backspace String Compare

### Problem

Character:

```text
#
```

acts as backspace.

Determine whether two strings become equal.

Example:

```text
ab#c

ad#c
```

Result:

```text
ac

ac
```

Answer:

```text
true
```

---

### Intuition

Treat typing like a text editor.

Normal character:

```text
Push
```

Backspace:

```text
Pop
```

---

### Algorithm

Build final strings using stack.

Compare results.

---

### Code

```java
class Solution {

    public boolean backspaceCompare(
            String s,
            String t) {

        return build(s).equals(build(t));
    }

    private String build(String str) {

        Stack<Character> st =
                new Stack<>();

        for(char ch : str.toCharArray()) {

            if(ch == '#') {

                if(!st.isEmpty()) {

                    st.pop();
                }
            }

            else {

                st.push(ch);
            }
        }

        StringBuilder ans =
                new StringBuilder();

        for(char ch : st) {

            ans.append(ch);
        }

        return ans.toString();
    }
}
```

### Complexity

```text
Time  : O(N)

Space : O(N)
```

# Variation 3

## LC735 - Asteroid Collision

### Problem

Positive asteroid:

```text
Moves Right
```

Negative asteroid:

```text
Moves Left
```

When opposite directions meet:

```text
Collision
```

Larger asteroid survives.

---

### Intuition

Only:

```text
Positive
followed by
Negative
```

can collide.

Stack stores active asteroids.

Whenever collision occurs:

```text
Compare Sizes

Destroy Smaller
```

---

### Code

```java
class Solution {

    public int[] asteroidCollision(
            int[] asteroids) {

        Stack<Integer> st =
                new Stack<>();

        for(int a : asteroids) {

            while(!st.isEmpty()
                    && st.peek() > 0
                    && a < 0
                    && st.peek() < -a) {

                st.pop();
            }

            if(!st.isEmpty()
                    && st.peek() > 0
                    && a < 0
                    && st.peek() == -a) {

                st.pop();
            }

            else if(st.isEmpty()
                    || st.peek() < 0
                    || a > 0) {

                st.push(a);
            }
        }

        int[] ans =
                new int[st.size()];

        for(int i=ans.length-1;i>=0;i--) {

            ans[i]=st.pop();
        }

        return ans;
    }
}
```

### Complexity

```text
Time  : O(N)

Space : O(N)
```

# Variation 4

## LC946 - Validate Stack Sequences

### Problem

Given:

```text
Push Order

Pop Order
```

Determine whether the pop sequence is possible.

---

### Intuition

Simulate exactly.

Push elements.

Whenever stack top equals next pop value:

```text
Pop
```

Continue until impossible.

---

### Code

```java
class Solution {

    public boolean validateStackSequences(
            int[] pushed,
            int[] popped) {

        Stack<Integer> st =
                new Stack<>();

        int j = 0;

        for(int num : pushed) {

            st.push(num);

            while(!st.isEmpty()
                    && st.peek() == popped[j]) {

                st.pop();
                j++;
            }
        }

        return st.isEmpty();
    }
}
```

### Complexity

```text
Time  : O(N)

Space : O(N)
```
