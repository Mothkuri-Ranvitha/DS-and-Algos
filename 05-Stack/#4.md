# Min Stack

## Definition

Min Stack problems involve maintaining the minimum element of a stack while supporting normal stack operations efficiently.

The challenge is:

```text
Push

Pop

Top

Get Minimum
```

all in:

```text
O(1)
```

time.

---

# Why Does This Pattern Exist?

Normally:

```java
stack = [5, 2, 8, 1]
```

To find minimum:

```text
Scan entire stack
```

Complexity:

```text
O(N)
```

Interviewers want:

```text
getMin()
```

in:

```text
O(1)
```

---

# Core Intuition

Store additional information while pushing.

Instead of storing only:

```text
Value
```

Store:

```text
Value

Current Minimum
```

for every position.

# LC155 - Min Stack

## Problem

Design a stack supporting:

```text
push()
pop()
top()
getMin()
```

All operations must run in:

```text
O(1)
```

---

## Intuition

Normal stack:

```text
[5,2,8,1]
```

Finding minimum requires:

```text
Scan entire stack
```

which is:

```text
O(N)
```

Instead, store:

```text
(value,currentMin)
```

for every element.

---

## Example

Push:

```text
5
```

Stack:

```text
(5,5)
```

---

Push:

```text
2
```

Stack:

```text
(5,5)
(2,2)
```

---

Push:

```text
8
```

Stack:

```text
(5,5)
(2,2)
(8,2)
```

Minimum is always available at top.

---

## Algorithm

Push:

```text
Store value and minimum so far
```

Pop:

```text
Normal pop
```

Top:

```text
Return value
```

GetMin:

```text
Return stored minimum
```

---

## Code

```java
class MinStack {

    Stack<int[]> st;

    public MinStack() {
        st = new Stack<>();
    }

    public void push(int val) {

        if(st.isEmpty()) {

            st.push(new int[]{val,val});
        }

        else {

            int min =
                Math.min(val, st.peek()[1]);

            st.push(new int[]{val,min});
        }
    }

    public void pop() {
        st.pop();
    }

    public int top() {
        return st.peek()[0];
    }

    public int getMin() {
        return st.peek()[1];
    }
}
```

---

## Complexity

```text
Push   O(1)

Pop    O(1)

Top    O(1)

GetMin O(1)

Space  O(N)
```

---

## Interview Takeaway

```text
Store current minimum
while pushing.
```

# Min Max Stack

## Goal

Support:

```text
getMin()

getMax()
```

simultaneously.

---

## Intuition

Store:

```text
(value,min,max)
```

for every element.

---

## Example

Push 5

```text
(5,5,5)
```

---

Push 2

```text
(2,2,5)
```

---

Push 8

```text
(8,2,8)
```

---

Push 1

```text
(1,1,8)
```

---

## Code

```java
class MinMaxStack {

    Stack<int[]> st =
            new Stack<>();

    public void push(int val) {

        if(st.isEmpty()) {

            st.push(new int[]{
                    val,val,val
            });
        }

        else {

            int min =
                Math.min(val,
                    st.peek()[1]);

            int max =
                Math.max(val,
                    st.peek()[2]);

            st.push(new int[]{
                    val,min,max
            });
        }
    }

    public void pop() {
        st.pop();
    }

    public int top() {
        return st.peek()[0];
    }

    public int getMin() {
        return st.peek()[1];
    }

    public int getMax() {
        return st.peek()[2];
    }
}
```

---

## Complexity

```text
Push   O(1)

Pop    O(1)

Top    O(1)

GetMin O(1)

GetMax O(1)

Space  O(N)
```

---

## Interview Takeaway

```text
Store both metadata values
while pushing.
```
# LC895 - Maximum Frequency Stack

## Problem

Design a stack that returns:

```text
Most Frequent Element
```

during pop.

If multiple elements have same frequency:

```text
Return most recent one.
```

---

## Example

Push:

```text
5
7
5
7
4
5
```

Frequencies:

```text
5 -> 3

7 -> 2

4 -> 1
```

Pop:

```text
5
```

because highest frequency.

---

## Core Intuition

Need two mappings.

### Frequency Map

```java
value -> frequency
```

Example:

```text
5 -> 3

7 -> 2
```

---

### Group Map

```java
frequency -> stack
```

Example:

```text
1 -> [5,7,4]

2 -> [5,7]

3 -> [5]
```

---

Highest frequency stack always contains answer.

---

## Code

```java
class FreqStack {

    Map<Integer,Integer> freq =
            new HashMap<>();

    Map<Integer,Stack<Integer>> group =
            new HashMap<>();

    int maxFreq = 0;

    public void push(int val) {

        int f =
            freq.getOrDefault(val,0)+1;

        freq.put(val,f);

        maxFreq =
            Math.max(maxFreq,f);

        group
            .computeIfAbsent(
                f,
                k -> new Stack<>()
            )
            .push(val);
    }

    public int pop() {

        int val =
            group.get(maxFreq).pop();

        freq.put(
            val,
            freq.get(val)-1
        );

        if(group.get(maxFreq)
                .isEmpty()) {

            maxFreq--;
        }

        return val;
    }
}
```

---

## Complexity

```text
Push O(1)

Pop  O(1)

Space O(N)
```

---

## Interview Takeaway

```text
Value → Frequency

Frequency → Stack

Track maximum frequency.
```

This is one of the most important advanced stack design problems.
